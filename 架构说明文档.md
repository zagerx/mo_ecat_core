# MoDriverPC_EtherCAT 架构说明文档

> 版本：v1.0  
> 目标：基于 SOEM 构建面向人型机器人、四足机器狗的 EtherCAT 主站，并提供 ROS2 接口。  
> 运行平台：Linux（x86_64 / ARM64）  
> 编程语言：C++17  
> 通信协议：EtherCAT + CiA402  

---

## 1. 项目背景与目标

### 1.1 背景

本项目 `MoDriverPC_EtherCAT` 作为上位机 EtherCAT 主站，用于驱动 Renesas 等支持 EtherCAT/CiA402 的伺服驱动器。典型应用场景包括：

- **人型机器人**：多自由度关节（双臂、双腿、躯干），轴数通常在 20~40 轴之间。
- **四足机器狗**：每条腿 3~4 个自由度，整机 12~16 轴。

这类系统对 EtherCAT 主站的要求：

1. **高实时性**：周期通常在 1ms ~ 4ms，部分高性能场景要求 500μs。
2. **多轴同步**：需要 Distributed Clock（DC）保证各关节时钟同步。
3. **高可靠性**：必须处理断线、从站故障、通信丢包等异常情况。
4. **ROS2 生态对接**：上层运动控制、轨迹规划、仿真通常基于 ROS2 开发。

### 1.2 目标

构建一个分层、模块化、可扩展的 EtherCAT 主站，使其能够：

- 通过标准 CiA402 协议控制各类伺服电机。
- 支持位置、速度、力矩、CSP/CSV/CST 等多种控制模式。
- 提供 ROS2 接口，便于与导航、运动规划、视觉等模块集成。
- 支持 YAML/JSON 配置，方便不同机器人型号快速适配。
- 具备良好的错误恢复能力（掉线重连、故障复位等）。

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| **分层解耦** | 硬件访问、协议解析、电机控制、机器人语义、ROS2 接口分层独立。 |
| **配置驱动** | 机器人轴数、PDO 映射、控制参数通过配置文件定义，避免硬编码。 |
| **实时与非实时分离** | EtherCAT 周期线程运行在实时优先级或独立线程；ROS2 接口运行在非实时上下文。 |
| **零拷贝数据共享** | EtherCAT 线程与 ROS2 线程通过 lock-free 环形缓冲区或原子变量共享数据。 |
| **故障安全** | 任何通信异常、状态异常都应及时进入安全状态（如触发 Quick Stop、切断使能）。 |
| **可测试** | 提供离线仿真模式、单元测试、示例程序。 |

### 3.2 模块职责

| 层级 | 模块 | 职责 |

| 电机控制层 | CiA402StateMachine | 处理 CiA402 状态转换：初始化 → 就绪 → 使能 → 运行 → 故障处理。 |
| 机器人语义层 | JointManager | 管理所有关节，维护关节名、ID、限位、方向、减速比等语义信息。 |

## 4. 核心模块详细设计

### 4.1 EtherCAT 主站层（EcMaster）

`EcMaster` 是对 SOEM 的 C++ 封装，向上层提供简洁的接口。

#### 关键接口

```cpp
class EcMaster {
public:
    bool Initialize(const std::string& ifname);
    void Close();

    bool ScanSlaves();
    int GetSlaveCount() const;
    const ec_slavet& GetSlave(int id) const;

    bool ConfigurePdo();
    bool StartCyclicThread(int cycle_time_us);
    bool StopCyclicThread();

    bool RequestOperationalState();
    bool RequestSafeOpState();
    bool RequestInitState();

    // 读写过程数据（PDO）
    void WriteOutput(int slave, int offset, const uint8_t* data, int len);
    void ReadInput(int slave, int offset, uint8_t* data, int len);

    // 邮箱通信（SDO）
    bool SdoRead(uint16_t slave, uint16_t index, uint8_t subindex,
                 void* data, int* len, int timeout_us);
    bool SdoWrite(uint16_t slave, uint16_t index, uint8_t subindex,
                  const void* data, int len, int timeout_us);

    // 获取运行时统计
    CyclicStats GetStats() const;

private:
    void CyclicTask();
    void StateMonitorTask();

    ecx_contextt ctx_;
    uint8_t iomap_[4096 * 4];  // 支持较多从站
    std::atomic<bool> running_{false};
    std::thread cyclic_thread_;
    std::thread monitor_thread_;
};
```

#### 线程模型

| 线程 | 优先级 | 周期 | 职责 |
|------|--------|------|------|
| Cyclic PDO Thread | 高（SCHED_FIFO） | 1ms（可配置） | 发送/接收 PDO，更新 IOmap，DC 同步。 |
| State Monitor Thread | 普通 | 10ms | 读取从站状态，检测掉线/故障，执行恢复。 |
| ROS2 Executor Thread | 普通 | 事件驱动 | 接收 ROS2 指令，发布状态。 |

#### 周期性任务流程

```cpp
void EcMaster::CyclicTask() {
    while (running_) {
        // 1. 发送过程数据
        ecx_send_processdata(&ctx_);

        // 2. 接收过程数据
        int wkc = ecx_receive_processdata(&ctx_, EC_TIMEOUTRET);

        // 3. 校验工作计数器
        if (wkc != expected_wkc_) {
            stats_.wkc_mismatch_count++;
        }

        // 4. DC 同步（若启用）
        if (use_dc_ && ctx_.slavelist[0].hasdc && wkc > 0) {
            SyncToDcClock();
        }

        // 5. 等待下一周期
        SleepUntilNextCycle();
    }
}
```

#### CiA402 状态机

```
Not Ready to Switch On
        ↓ 上电自检完成
Switch On Disabled
        ↓ CW=0x06 (Shutdown)
Ready to Switch On
        ↓ CW=0x07 (Switch On)
Switched On
        ↓ CW=0x0F (Enable Operation)
Operation Enabled  ←── 正常运行
        ↓
   Quick Stop Active  ←── CW=0x02 (Quick Stop)
        ↓
   Switch On Disabled
```

安全监控在 PDO 周期线程中执行，一旦触发异常：

1. 立即向所有驱动器发送 Quick Stop 命令。
2. 通知 ROS2 节点发布紧急状态。
3. 记录故障码和时间戳。

## 5. ROS2 接口设计

### 5.1 节点设计

创建一个 ROS2 节点 `ethercat_master_node`，负责：

- 订阅关节指令。
- 发布关节状态。
- 提供使能、禁用、复位、回零等服务。
- 发布诊断信息。

### 5.2 Topic 设计

| Topic 名称 | 类型 | 方向 | 说明 |
|------------|------|------|------|
| `/joint_states` | `sensor_msgs/JointState` | 发布 | 各关节实际位置、速度、力矩。 |
| `/joint_commands` | `sensor_msgs/JointState` 或自定义 `JointCommand` | 订阅 | 目标位置/速度/力矩。 |
| `/ecat_diagnostics` | `diagnostic_msgs/DiagnosticArray` | 发布 | EtherCAT 通信状态、从站状态。 |
| `/ecat_master_status` | 自定义 `EcatMasterStatus` | 发布 | 主站运行状态、周期抖动、WKC 错误计数。 |

### 5.3 Service 设计

| Service 名称 | 类型 | 说明 |
|--------------|------|------|
| `/enable` | `std_srvs/Trigger` | 使能所有关节。 |
| `/disable` | `std_srvs/Trigger` | 禁用所有关节。 |
| `/reset_fault` | `std_srvs/Trigger` | 清除所有关节故障。 |
| `/home` | `std_srvs/Trigger` | 执行回零操作。 |
| `/set_operation_mode` | 自定义 | 切换控制模式（位置/速度/力矩）。 |

### 5.4 Action 设计

| Action 名称 | 类型 | 说明 |
|-------------|------|------|
| `/follow_joint_trajectory` | `control_msgs/FollowJointTrajectory` | 执行多关节轨迹，与 MoveIt2 兼容。 |

### 5.5 ROS2 与实时线程的数据交互

为了避免 ROS2 非实时线程阻塞 EtherCAT 实时线程，采用 **双缓冲（Double Buffering）** 机制：

```text
EtherCAT Cyclic Thread          ROS2 Executor Thread
        │                                │
        ▼                                ▼
┌───────────────┐                ┌───────────────┐
│  Write to     │◄──────────────►│  Read from    │
│  Buffer A     │   atomic swap  │  Buffer B     │
└───────────────┘                └───────────────┘
```

- EtherCAT 线程每周期把关节状态写入当前写缓冲。
- ROS2 线程定时读取当前读缓冲。
- 通过 `std::atomic` 指针交换实现无锁切换。

## 6. 实时性设计

### 6.1 线程优先级

```cpp
void SetRealtimePriority(std::thread& thread, int priority) {
    sched_param param{};
    param.sched_priority = priority;
    pthread_setschedparam(thread.native_handle(), SCHED_FIFO, &param);
}
```

| 线程 | 调度策略 | 优先级 | 说明 |
|------|----------|--------|------|
| Cyclic PDO Thread | SCHED_FIFO | 80 | 最高优先级，保证周期稳定。 |
| State Monitor Thread | SCHED_OTHER | 0 | 普通优先级。 |
| ROS2 Executor | SCHED_OTHER | 0 | 普通优先级。 |

### 6.2 抖动监控

在周期线程中记录：

- 最小/最大/平均周期时间。
- 超周期次数（实际周期 > 设定周期 1.5 倍）。
- WKC 不匹配次数。

这些数据通过 `/ecat_master_status` Topic 发布。

### 6.3 周期时间选择

| 应用场景 | 建议周期 | 说明 |
|----------|----------|------|
| 人型机器人全身控制 | 1ms ~ 2ms | 平衡实时性与 CPU 负载。 |
| 四足机器狗力控 | 1ms | 力控回路需要较快更新。 |
| 高动态操作臂 | 500μs ~ 1ms | 需要更高带宽。 |

## 8. 安全与故障处理

### 8.1 故障分级

| 级别 | 名称 | 处理方式 |
|------|------|----------|
| 0 | Info | 记录日志，继续运行。 |
| 1 | Warning | 记录日志，通知上层。 |
| 2 | Error | 尝试自动恢复（如重新使能）。 |
| 3 | Critical | 立即触发 Quick Stop / Disable，停止运动。 |

### 8.2 常见故障与恢复策略

| 故障 | 检测方式 | 恢复策略 |
|------|----------|----------|
| 从站掉线 | WKC 不匹配 + readstate == NONE | `ecx_recover_slave()` 尝试恢复。 |
| 驱动器报警 | Status Word 故障位 | `ResetFault()` → 重新使能。 |
| 关节超限 | SafetyMonitor 检查 | 触发 Quick Stop，通知上层。 |
| 通信超时 | 连续 missed cycles | 进入 SAFE_OP，记录故障。 |
| 主站启动失败 | init/config 失败 | 返回错误，不进入 OP。 |

### 8.3 紧急停止

紧急停止信号应通过两种方式传播：

1. **软件急停**：ROS2 Service `/disable` 或 SafetyMonitor 触发 Quick Stop。
2. **硬件急停**：通过数字 IO（如 SOEM GPIO 扩展）或驱动器 STO 安全转矩关闭接口实现。

## 11. 关键技术决策

### 11.1 为什么使用 SOEM？

- 开源、轻量、跨平台（Linux/Windows/RTOS）。
- 社区成熟，示例丰富，文档完善。
- 支持 DC 时钟同步，满足多轴同步需求。

### 11.2 为什么用 C++17？

- 面向对象设计适合封装复杂的机器人和电机语义。
- 与 ROS2（rclcpp）天然兼容。
- 现代 C++ 特性（智能指针、线程库、optional 等）提高开发效率。

### 11.3 为什么 ROS2 与实时线程分离？

- ROS2 默认不使用实时调度，避免阻塞 EtherCAT 周期线程。
- 双缓冲机制保证数据交换无锁、低延迟。

### 11.4 是否使用实时内核？

- **开发阶段**：普通 Linux 内核即可，配合 SCHED_FIFO 已能满足 1ms 周期。
- **量产/高动态场景**：建议切换为 **PREEMPT_RT** 实时内核，进一步降低抖动。

---

## 12. 注意事项

1. **root 权限**：Linux 下 SOEM 需要 root 或 `CAP_NET_RAW` 能力才能发送原始以太网帧。
2. **网卡选择**：务必使用独立网卡作为 EtherCAT 接口，避免与普通网络混用。
3. **ESI 文件**：不同驱动器的 PDO 映射差异很大，需根据 ESI 文件或 `slaveinfo -map` 输出配置。
4. **DC 配置**：多轴同步时务必启用 DC，并正确配置同步周期。
5. **安全第一**：所有使能、运动指令都应经过安全层检查，避免机械损坏。
