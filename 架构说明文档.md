# MoDriverPC_EtherCAT 架构说明文档

> 版本：v2.1（增加从站类型识别）  
> 目标：基于 SOEM 构建面向多从站 EtherCAT 主站，后续可扩展机器人关节控制。  
> 运行平台：Linux（x86_64 / ARM64）  
> 编程语言：C++17  
> 通信协议：EtherCAT

---

## 1. 项目目标

本项目作为上位机 EtherCAT 主站，用于驱动支持 EtherCAT 的伺服驱动器、IO 模块等设备。当前阶段重点：**管理多个从站的生命周期和 PDO 数据交换**，CiA402 状态机等电机控制逻辑后续根据实际驱动器逐步添加。

---

## 2. 设计原则

| 原则 | 说明 |
|------|------|
| **分层解耦** | 硬件访问、总线通信、从站管理、电机语义分层独立。 |
| **一从站一节点** | 每个扫描到的从站对应一个 `SlaveNode`，独立管理通信状态、PDO 映射和数据。 |
| **电机语义与通信分离** | `ServoAxis` 只描述电机命令/反馈，不感知 EtherCAT/PDO/SDO。 |
| **线程管理在上层** | 周期线程、监控线程由 `main` 创建和管理，`EcatController` 只提供周期步进函数。 |
| **异步日志** | 日志系统不阻塞 EtherCAT 周期线程。 |
| **不兼容旧代码** | 旧设计中不合理的内容已删除，当前文档只描述当前代码。 |

## 4. 核心模块职责

### 4.1 EcMaster（EtherCAT 主站层）

`src/ec_master/ec_master.h`

职责：

- 初始化 SOEM、绑定网卡
- 扫描从站、配置 PDO 映射、DC 时钟
- 全局/单站 EtherCAT 状态切换
- PDO 收发
- SDO 读写接口（当前为框架，功能后续补齐）

关键接口：

```cpp
class EcMaster {
public:
    bool Initialize(const EcMasterConfig &config);
    void Close();

    bool ScanAndConfigure();
    int GetSlaveCount() const;

    // 全局状态切换
    bool RequestOperationalState();
    bool RequestSafeOpState();
    bool RequestInitState();

    // 单站状态切换
    bool RequestState(int slave, uint16_t state);
    uint16_t GetCurrentState(int slave) const;

    // PDO 周期收发
    void RunOneCycle();

    // 从站状态检查
    void CheckSlaveStates();

    // PDO 字节偏移读写（由 SlaveNode 使用）
    void WriteOutput(int slave, int offset, const uint8_t *data, int len);
    void ReadInput(int slave, int offset, uint8_t *data, int len);

    // SDO 框架接口
    bool SdoRead(uint16_t slave, uint16_t index, uint8_t subindex,
                 void *data, int len, int timeout_us);
    bool SdoWrite(uint16_t slave, uint16_t index, uint8_t subindex,
                  const void *data, int len, int timeout_us);
};
```

> **注意**：Init / PreOP / SafeOP / OP 四个 EtherCAT 状态由 `EcMaster` 管理，`SlaveNode` 可以请求单个从站的状态切换，但底层切换能力来自 `EcMaster`。

---

### 4.2 EcatController（总线应用控制层）

`src/ec_controller/ec_controller.h`

职责：

- 封装总线级生命周期：Initialize → StartOperation → Stop
- 持有 `EcMaster` 和 `SlaveNodeManager`
- 提供 `RunOneCycle()` 给 PDO 周期线程调用
- 提供 `GetSlaveNodeManager()` 给上层访问从站

```cpp
class EcatController {
public:
    bool Initialize(const EcMasterConfig &config);
    bool StartOperation();
    void Stop();

    void RunOneCycle();
    void CheckSlaveStates();

    SlaveNodeManager &GetSlaveNodeManager();
};
```

周期流程：

```cpp
void EcatController::RunOneCycle() {
    slave_node_manager_.UpdateAllOutputs();  // 所有 SlaveNode：ServoAxis → IOmap
    master_.RunOneCycle();                    // PDO 收发
    slave_node_manager_.UpdateAllInputs();   // 所有 SlaveNode：IOmap → ServoAxis
}
```

### 4.3 SlaveNode（从站节点）

`src/slave_node/slave_node.h`

每个扫描到的从站对应一个 `SlaveNode`。它负责：

- 维护本从站的 EtherCAT 通信状态
- 维护本从站的 PDO 映射
- 提供 SDO 读写能力（框架）
- 挂载对应的 `DeviceProfile`，在周期循环中把业务语义量编码成 PDO 字节，或把 PDO 字节解码成语义量

```cpp
class SlaveNode {
public:
    SlaveNode(EcMaster &master, const SlaveConfig &config, const SlaveInfo &info);

    bool RequestState(uint16_t state);
    uint16_t GetCurrentState() const;

    bool SdoRead(uint16_t index, uint8_t subindex, void *data, size_t len, int timeout_us);
    bool SdoWrite(uint16_t index, uint8_t subindex, const void *data, size_t len, int timeout_us);

    bool ConfigurePdoMapping();
    void SetPdoMapping(const PdoMapping &mapping);

    void UpdatePdoOutput();
    void UpdatePdoInput();

    SlaveType GetType() const;
    DeviceProfile *GetProfile() const;
    ServoAxis *GetServoAxis() const; // 仅 Servo 类型返回非空
};
```

`SlaveConfig` / `PdoMapping`：

```cpp
enum class SlaveType { Unknown, Servo, DigitalIo, AnalogIo };

struct PdoEntry {
    std::string name;
    int offset;
    int size;
};

struct PdoMapping {
    int output_bytes = 0;
    int input_bytes = 0;
    std::vector<PdoEntry> outputs;
    std::vector<PdoEntry> inputs;
};

struct SlaveConfig {
    SlaveType type = SlaveType::Unknown;
    PdoMapping mapping;
    ServoAxis::Config axis_config;
};
```

当前 `ServoProfile` 的默认 PDO 映射只包含位置/速度/力矩数据条目，**不含 CiA402 控制字/状态字**；周期更新时按 `axis_config` 中的缩放系数进行原始值与工程单位的转换。

非伺服设备（`DigitalIoProfile`、`AnalogIoProfile`、`UnknownProfile`）当前为框架实现，后续根据实际硬件扩展 PDO 条目与业务语义。

### 4.4 SlaveNodeManager（从站节点管理器）

`src/slave_node/slave_node_manager.h`

职责：

- 根据从站数量创建所有 `SlaveNode`
- 通过 `SlaveTypeDetector` 识别设备类型，通过 `SlaveConfigFactory` 生成对应配置
- 按索引/名字查找从站
- 批量更新所有从站 PDO
- 批量请求状态切换

```cpp
class SlaveNodeManager {
public:
    bool Initialize(EcMaster &master, int slave_count);

    size_t GetNodeCount() const;
    SlaveNode *GetNode(size_t index);
    SlaveNode *GetNode(const std::string &name);

    void UpdateAllOutputs();
    void UpdateAllInputs();
    bool RequestAllState(uint16_t state);

    void Clear();
};
```

### 4.5 ServoAxis（电机语义对象）

`src/servo_axis/servo_axis.h`

纯电机语义对象，**不感知 EtherCAT / PDO / SDO**。

```cpp
class ServoAxis {
public:
    struct Config {
        std::string name;
        double position_scale;
        double velocity_scale;
        double torque_scale;
        int direction;
    };

    // 上层命令
    void SetTargetPosition(double rad);
    void SetTargetVelocity(double rad_per_s);
    void SetTargetTorque(double nm);
    void SetEnable(bool enable);

    // 上层反馈
    double GetActualPosition() const;
    double GetActualVelocity() const;
    double GetActualTorque() const;
    bool IsEnabled() const;
    bool HasFault() const;

    // 由 SlaveNode 周期更新
    void UpdateCommandReadback(double position, double velocity, double torque,
                               bool enabled, bool fault);
};
```

后续 CiA402 状态机可以在 `ServoAxis` 基础上再包一层实现，不影响现有框架。

### 4.6 CyclicRunner（通用周期任务调度器）

`src/cyclic/cyclic_runner.h`

独立的周期线程封装：

- 按固定周期调用用户任务
- 支持设置实时优先级、CPU 亲和性
- 基于绝对时间点的 `sleep_until`，减少周期漂移

> 由 `main` 创建并管理，不绑定任何业务逻辑。

---

### 4.7 Logger（异步日志）

`src/utils/logger.h`

- 五级日志：Debug / Info / Warn / Error / Fatal
- 输出带时间戳、级别、源文件行号
- 默认输出到 `logger/ecat.log`，自动创建目录
- **异步**：调用方只入队，后台线程负责写入，不阻塞业务线程
- 支持编译时日志级别裁剪

## 5. 线程模型

```text
main 线程
    ├── 创建 PDO 周期线程（高优先级，默认 1ms）
    │       └── EcatController::RunOneCycle()
    │               ├── SlaveNodeManager::UpdateAllOutputs()
    │               ├── EcMaster::RunOneCycle()
    │               └── SlaveNodeManager::UpdateAllInputs()
    │
    ├── 创建状态监控线程（普通优先级，默认 10ms）
    │       └── EcatController::CheckSlaveStates()
    │
    ├── 注册退出信号
    └── WaitForShutdown()
```

说明：

- 周期线程和状态监控线程访问同一个 SOEM 上下文，由 `EcMaster` 内部互斥锁保护。
- 实时优先级、CPU 亲和性由 `main` 通过 `CyclicRunner` 设置，不由库内部决定。

```text
上层应用
    │

    │

    │

```

## 9. 关键技术决策

### 9.1 为什么把线程管理放在 main？

实时优先级、调度策略、CPU 亲和性属于系统级配置，应该由应用层决定，库不应该私自创建高优先级线程。

### 9.2 为什么 ServoAxis 不直接操作 PDO？

把“电机语义”和“通信细节”分离后：

- 更换驱动器或 PDO 映射时，`ServoAxis` 不需要改。
- 后续加 CiA402 状态机时，不会影响现有的 PDO 数据桥接。

### 9.3 为什么把 `DeviceProfile` 从 `SlaveNode` 中拆分出来？

`SlaveNode` 负责 EtherCAT 通信细节（PDO 映射、状态、SDO），而 `DeviceProfile` 负责具体设备语义（伺服、数字 IO、模拟 IO）。拆分后：

- 新增设备类型时无需改动 `SlaveNode` 的通信逻辑。
- 伺服电机的 `ServoAxis` 不再被强制挂载到 IO 模块等无关设备上。
- 同一类设备可复用同一个 `DeviceProfile` 实现。

### 9.4 为什么一从站一 SlaveNode？

每个从站的 PDO 映射、EtherCAT 状态、SDO 参数都可能不同。独立节点后：

- 可以单独控制某个从站。
- 可以单独更换某个从站的配置。
- 多从站代码清晰、可维护。
