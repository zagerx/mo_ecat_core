# 当前问题与解决思路

## 当前目标

搭建健壮的 EtherCAT 主站初始化流程，确保从站稳定进入 **PREOP** 状态并支持 **SDO 通信**。当前阶段不涉及 PDO 通信。

---

## 当前存在的主要问题

### 1. 初始化流程超出了当前目标范围

当前 `EcatController::Initialize()` 会走到 `kDcConfigured`：

```text
InitDone → Scanned → PreOp → PdoConfigured → SafeOp → DcConfigured
```

按当前目标，初始化应只到 **PreOp** 就结束。后续 `PdoConfigured / SafeOp / DcConfigured / Operational` 是 PDO 阶段的事情。

### 2. `main.cpp` 请求了 OPERATIONAL 状态

当前 `main.cpp` 调用 `controller.StartOperation()`，会请求从站进入 OP。这与当前目标冲突。

### 3. 状态机粒度与当前目标不匹配

`ControllerState` 枚举虽然完整，但 `Initialize()` 实际走到的状态过深。需要让状态机清晰区分：
- **初始化完成态**：`kPreOp`
- **后续 PDO 阶段状态**：`kPdoConfigured / kSafeOp / kDcConfigured / kOperational`

### 4. `SlaveConfigFactory` 的硬编码映射与 RA8 不符

`MakeServoConfig()` 假设 10 字节输出/输入（位置/速度/力矩），但 RA8 实际 ESI 中是 4 字节 `OutputCounter` / 4 字节 `InputCounter`。

虽然当前不做 PDO 通信，但这个硬编码映射会在创建 `SlaveNode` 时分配 buffer，可能给后续带来隐患。

### 5. 工作区存在未提交修改

```text
 M src/ec_controller/ec_controller.cpp
 M src/ec_controller/ec_controller.h
 M src/ec_master/ec_master.cpp
 M src/ec_master/ec_master.h
 M src/main.cpp
?? docs/RA8 EtherCAT.xml
```

这些修改已经部分实现了状态机，但还未整理提交。

## 解决思路

### 思路 1：缩短初始化流程到 PREOP

修改 `EcatController::Initialize()`：

```cpp
bool EcatController::Initialize(const EcMasterConfig &config)
{
    if (initialized_) { ... }

    if (!master_.Initialize(config)) { return false; }
    state_ = ControllerState::kInitDone;

    auto slave_infos = master_.ScanSlaves();
    if (slave_infos.empty()) { Shutdown(false); return false; }
    state_ = ControllerState::kScanned;

    if (!master_.RequestPreOpState()) { ... }
    if (!master_.CheckAllSlavesInState(EC_STATE_PRE_OP)) { ... }
    state_ = ControllerState::kPreOp;

    // 创建 SlaveNode，仅用于管理从站信息，不涉及 PDO 配置
    if (!node_manager_.Initialize(master_, slave_infos)) { ... }

    // 初始化完成，SDO 可用
    initialized_ = true;
    return true;
}
```

移除 `ConfigureProcessData()`、`ConfigureDc()`、`CheckAllSlavesInState(EC_STATE_SAFE_OP)` 等 PDO 相关步骤。

### 思路 2：调整 `main.cpp`

移除 `StartOperation()` 调用：

```cpp
int main(int argc, char *argv[])
{
    ...
    if (!controller.Initialize(config)) {
        return 1;
    }

    LOG_INFO << "System initialized, slaves in PREOP, SDO ready";

    // 暂时不进入 OPERATIONAL，PDO 阶段再启用
    // controller.StartOperation();

    RegisterShutdownSignals();
    WaitForShutdown();
    controller.Stop();
    return 0;
}
```

### 思路 3：保留完整状态机枚举，但分阶段使用

`ControllerState` 可以保留完整枚举，但明确各阶段边界：

```cpp
enum class ControllerState {
    kUninitialized,
    kInitDone,
    kScanned,
    kPreOp,          // 当前阶段目标：初始化完成，SDO 可用
    kPdoConfigured,  // 后续 PDO 阶段
    kSafeOp,         // 后续 PDO 阶段
    kDcConfigured,   // 后续 PDO 阶段
    kOperational,    // 后续 PDO 阶段
    kError
};
```

### 思路 4：可选的 SDO 验证

初始化完成后，可以通过 SDO 读取从站对象字典来验证通信：

```cpp
uint32_t device_type;
master_.SdoRead(1, 0x1000, 0, &device_type, sizeof(device_type), 10000);
```

这是调试手段，不是必须。

### 思路 5：后续 PDO 阶段的入口预留

当前先不实现，但预留清晰的扩展点：

- `ConfigureProcessData()`：调用 SOEM `ecx_config_map_group()` 配置 IOmap
- `ConfigureDc()`：配置 Distributed Clock
- `StartOperation()`：请求 OP 并启动 PDO 周期线程

这些函数已经存在，当前只是不在 `Initialize()` 中调用。
