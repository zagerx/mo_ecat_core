# 当前问题与解决思路

## 当前目标

在主站状态机框架已完成、从站能稳定进入 **PREOP**、**SDO 通信已验证**的基础上，进一步**扩展 Activity 框架**并**补强状态机健壮性**。当前阶段仍不进入 OP，PDO 映射等应用层细节延后处理。

---

## 最近进展

### 1. SDO 通信验证已完成

- `EcMaster::SdoRead()` / `SlaveNode::SdoRead()` 已通过实际硬件验证。
- 首个 Activity `SdoDiagnosticsActivity` 已实现，在 `main.cpp` 中对每个从站读取：
  - `0x1000:00` Device type
  - `0x1018:01` Vendor ID
  - `0x1018:02` Product Code
  - `0x1018:03` Revision Number
- 日志已能正确输出读取结果（如 `vendor_id=0x766`、`product_code=0x2000` 等）。
- SDO 验证不再放在 `EcatController::Initialize()` 内部强制执行，而是通过 Activity 机制灵活触发。

### 2. 主站状态机框架已稳定

- `EcatController` 内建线性生命周期状态机（`kUninitialized → … → kOperational → kError`）。
- `TransitionTo(ControllerState target)` 统一入口，支持自动顺序前进和后退转换。
- `kAllowedTransitions` 状态转换表约束非法跳转，前进路径也已查表校验。
- `DoInit() / DoScan() / DoPreOp() / DoPdoConfigure() / DoSafeOp() / DoDcConfigure() / DoOperational() / DoShutdown()` 已拆分。
- `Initialize() / StartOperation() / Stop()` 统一调用 `TransitionTo()`。
- `EnterErrorState()` 已实现，`CheckSlaveStates()` 在 OP 阶段检测到掉线时进入 `kError`。
- `DoShutdown()` 根据当前 `state_` 精确回滚。

### 3. Activity 同步框架已落地

- `EcatActivity` 基类、`ActivityFailurePolicy`、`EcatController::ExecuteActivity()` 已实现。
- `activity_running_` 保证同一时刻只有一个 Activity 在运行。
- `SdoDiagnosticsActivity` 作为首个低风险 Activity 验证了整个框架。

### 4. 从站节点层已完成占位重构

- `SlaveNode` 两阶段生命周期：占位阶段支持 SDO/状态查询，配置阶段再分配 PDO buffer。
- `SlaveNodeManager` 管理占位节点，支持按索引/名字查找。

---

## 当前代码下的问题

### 1. Activity 体系尚未充分扩展

当前只有 `SdoDiagnosticsActivity` 一个只读 Activity。框架是否能支撑"写操作"、"状态巡检"、"单站状态切换"等场景，还需要通过实际 Activity 验证。

### 2. `CheckSlaveStates()` 监控能力有限

当前只在 `kOperational` 时检测从站是否掉出 OP。在 `kPreOp` / `kSafeOp` 等中间状态下，从站掉线或进入 ERROR 状态不会被主站状态机感知，不利于早期异常发现和恢复。

### 3. 缺少单元测试

项目目前没有任何测试代码。`EcatController` 状态转换、`ActivityFailurePolicy` 处理、`SlaveTypeDetector` 等逻辑都依赖手动上电验证，回归成本高。

### 4. DC 失败策略尚未明确

`DoDcConfigure()` 当前只要失败就进入 `kError`。`use_dc == false` 时是否跳过、部分从站不支持 DC 时是否允许跳过，还需要根据实际需求明确并文档化。

### 5. PDO 映射硬编码问题已延后

`SlaveConfigFactory::MakeServoConfig()` 仍生成 10 字节位置/速度/力矩映射，与 RA8 实际 4 字节 `OutputCounter/InputCounter` 不符。该问题不影响当前框架搭建，待框架稳定后再通过动态 PDO 映射读取解决。
