# 当前问题与解决思路

## 当前目标

构建健壮、可扩展的 EtherCAT 主站状态机框架，确保从站稳定进入 **PREOP** 并支持 **SDO**。当前阶段不涉及 PDO 通信。

---

## 最近进展

### SlaveNode / SlaveNodeManager 占位节点重构已完成

- `SlaveNode` 构造函数改为只接收 `EcMaster &` 和 `SlaveInfo &`，创建占位节点。
- 新增 `Configure(const SlaveConfig &)`，在 PDO 阶段再分配 PDO buffer、创建 `DeviceProfile`。
- `SlaveNodeManager::Initialize()` 只创建占位节点；新增 `ConfigureAll()` 供 PDO 阶段调用。
- `UpdateAllOutputs/Inputs()` 自动跳过未配置节点。
- PREOP 阶段不再生成硬编码 PDO 映射，10 字节伺服映射与 RA8 实际 4 字节不符的问题已延迟到 PDO 阶段解决。

---

## 当前代码下的问题

### 1. 没有统一的状态转换入口

`Initialize()`、`StartOperation()`、`Stop()` 中各自维护状态转换逻辑，导致：
- 前置条件检查分散
- 失败回滚路径不一致
- 新增状态时容易遗漏处理
- 难以单元测试

### 2. 缺少状态转换约束

当前代码没有任何机制防止非法跳转，例如：
- 从 `kUninitialized` 直接跳到 `kOperational`
- 从 `kOperational` 直接跳回 `kPreOp` 而不经过 SAFEOP

### 3. `operational_` 和 `state_` 冗余

`operational_` 是 `state_ == kOperational` 的缓存，但维护两个变量容易不一致：
- `StartOperation()` 失败时 `operational_` 未被重置
- 运行中从站掉出 OP 时 `operational_` 不会自动更新

### 4. `StartOperation()` 不完整

请求 OP 后直接置 `operational_ = true`，没有确认所有从站真的到达 OP：

```cpp
if (!master_.RequestOperationalState()) { return false; }
operational_ = true;
```

### 5. `Stop()` / `Shutdown()` 回滚路径不精确

`Shutdown(true)` 无条件请求 SAFE_OP 再请求 INIT。如果当前只是 PREOP，请求 SAFE_OP 是多余的，应该根据 `state_` 决定回滚步骤。

### 6. `kError` 状态未使用

`ControllerState::kError` 枚举存在，但没有任何代码会设置它。运行中状态异常时无法表达，也无法触发错误恢复。

### 7. `Initialize()` 中 SlaveInfo 打印冗余

`ScanSlaves()` 和 `RefreshSlaveInfos()` 后各打印一次完整 SlaveInfo，日志重复。

### 8. `CheckSlaveStates()` 没有实际监控作用

当前只是读取一次从站状态，不判断、不报警、不降级。后续 PDO 阶段需要真正的状态监控能力。

## 解决思路

### 思路 1：统一状态转换入口

引入 `TransitionTo(ControllerState target)`：

```cpp
bool EcatController::TransitionTo(ControllerState target);
```

对外接口改为：

```cpp
bool Initialize(const EcMasterConfig &config) {
    config_ = config;
    return TransitionTo(ControllerState::kPreOp);
}

bool StartOperation() {
    return TransitionTo(ControllerState::kOperational);
}

void Stop() {
    TransitionTo(ControllerState::kUninitialized);
}
```

### 思路 2：定义状态转换表

```cpp
static const std::map<ControllerState, std::vector<ControllerState>> kAllowedTransitions = {
    {kUninitialized, {kInitDone}},
    {kInitDone,      {kScanned, kUninitialized}},
    {kScanned,       {kPreOp, kUninitialized}},
    {kPreOp,         {kPdoConfigured, kUninitialized}},
    {kPdoConfigured, {kSafeOp, kPreOp, kUninitialized}},
    {kSafeOp,        {kDcConfigured, kPreOp, kUninitialized}},
    {kDcConfigured,  {kOperational, kSafeOp, kUninitialized}},
    {kOperational,   {kSafeOp, kUninitialized}},
    {kError,         {kUninitialized}}
};
```

`TransitionTo()` 先校验合法性：

```cpp
bool EcatController::TransitionTo(ControllerState target)
{
    auto it = kAllowedTransitions.find(state_);
    if (it == kAllowedTransitions.end() ||
        std::find(it->second.begin(), it->second.end(), target) == it->second.end()) {
        LOG_ERROR << "Invalid transition: " << StateToString(state_)
                  << " -> " << StateToString(target);
        return false;
    }
    // 执行转换...
}
```

### 思路 3：拆分状态转换实现

每个状态对应一个私有方法：

```cpp
bool DoInit();
bool DoScan();
bool DoPreOp();
bool DoPdoConfigure();   // 后续 PDO 阶段
bool DoSafeOp();         // 后续 PDO 阶段
bool DoDcConfigure();    // 后续 PDO 阶段
bool DoOperational();    // 后续 PDO 阶段
bool DoShutdown();
bool DoError();
```

`TransitionTo()` 根据目标状态调用对应方法。

### 思路 4：明确每个状态的前置条件和失败策略

```cpp
bool EcatController::DoPreOp()
{
    if (state_ != ControllerState::kScanned) {
        LOG_ERROR << "Invalid state for PreOp: " << StateToString(state_);
        return false;
    }

    if (!master_.RequestPreOpState()) {
        return false;
    }

    if (!master_.CheckAllSlavesInState(EC_STATE_PRE_OP)) {
        master_.RequestInitState();
        return false;
    }

    state_ = ControllerState::kPreOp;
    return true;
}
```

### 思路 5：清理 `operational_`

去掉 `operational_` 成员：

```cpp
bool IsOperational() const {
    return state_ == ControllerState::kOperational;
}
```

### 思路 6：精确的回滚路径

```cpp
bool EcatController::DoShutdown()
{
    if (state_ >= ControllerState::kOperational) {
        master_.RequestSafeOpState();
    }
    if (state_ >= ControllerState::kPreOp) {
        master_.RequestInitState();
    }

    node_manager_.Clear();
    master_.Close();
    state_ = ControllerState::kUninitialized;
    initialized_ = false;
    return true;
}
```

### 思路 7：实现 `kError` 状态

```cpp
void EcatController::EnterErrorState(const std::string &reason)
{
    LOG_ERROR << "Entering error state: " << reason;
    state_ = ControllerState::kError;
}
```

运行中监控：

```cpp
void EcatController::CheckSlaveStates()
{
    master_.CheckSlaveStates();
    if (state_ == ControllerState::kOperational) {
        if (!master_.CheckAllSlavesInState(EC_STATE_OPERATIONAL)) {
            EnterErrorState("Slave dropped out of OPERATIONAL");
        }
    }
}
```

### 思路 8：简化日志

只在关键状态转换点打印 SlaveInfo，PREOP 后的刷新只打印关键字段或只打印 summary。
