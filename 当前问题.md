# 当前问题与解决思路

## 当前目标

在已完成占位节点重构和主站状态机框架的基础上，进一步验证 **SDO 通信**，并为后续 PDO 阶段实现动态 PDO 映射读取打好基础。当前阶段仍不进入 OP。

---

## 最近进展

### 1. SlaveNode / SlaveNodeManager 占位节点重构已完成

- `SlaveNode` 构造函数改为只接收 `EcMaster &` 和 `SlaveInfo &`，创建占位节点。
- 新增 `Configure(const SlaveConfig &)`，在 PDO 阶段再分配 PDO buffer、创建 `DeviceProfile`。
- `SlaveNodeManager::Initialize()` 只创建占位节点；新增 `ConfigureAll()` 供 PDO 阶段调用。
- `UpdateAllOutputs/Inputs()` 自动跳过未配置节点。
- PREOP 阶段不再生成硬编码 PDO 映射，10 字节伺服映射与 RA8 实际 4 字节不符的问题已延迟到 PDO 阶段解决。

### 2. EcatController 状态机框架重构已完成

- 引入统一状态转换入口 `TransitionTo(ControllerState target)`。
- 定义 `kAllowedTransitions` 状态转换表，防止非法状态跳转。
- 拆分 `DoInit() / DoScan() / DoPreOp() / DoPdoConfigure() / DoSafeOp() / DoDcConfigure() / DoOperational() / DoShutdown()`。
- `Initialize() / StartOperation() / Stop()` 统一调用 `TransitionTo()`。
- 移除 `operational_` 标志，`IsOperational()` 基于 `state_` 判断。
- 实现 `EnterErrorState()`，`CheckSlaveStates()` 在 OP 阶段检测到掉线时进入 `kError`。
- `DoShutdown()` 根据当前 `state_` 精确回滚，PREOP 直接回 INIT，不请求 SAFEOP。
- 简化日志：扫描后打印一次完整 SlaveInfo，进入 PREOP 后只打印 summary。

---

## 当前代码下的问题

### 1. SDO 通信尚未验证

`EcMaster::SdoRead()` / `SdoRead()` 已实现 SOEM 调用，但 `EcatController` 初始化流程中还没有用 SDO 读取从站对象字典来验证通信。

建议验证项：
- `0x1000:00` Device type
- `0x1018:01` Vendor ID
- `0x1018:02` Product Code
- `0x1018:03` Revision Number

### 2. PDO 映射仍依赖硬编码配置

`SlaveConfigFactory::MakeServoConfig()` 仍生成 10 字节位置/速度/力矩映射，与 RA8 实际 4 字节 `OutputCounter/InputCounter` 不符。

占位节点重构后，映射生成已延迟到 PDO 阶段；后续需要通过 SDO 读取从站 PDO 映射并替换硬编码配置。

### 3. `CheckSlaveStates()` 监控能力有限

当前只在 `kOperational` 时检测从站是否掉出 OP。PREOP/SAFEOP 等中间状态没有统一的状态异常处理，错误恢复路径也尚未完整验证。

### 4. `EcatController` 尚未提供公共的 SDO 访问接口

上层应用目前只能通过 `GetSlaveNodeManager()` 获取 `SlaveNode` 后再调用 `SdoRead/SdoWrite`。是否需要在 `EcatController` 层提供批量/便捷的 SDO 接口，视后续应用需求决定。

## 下一阶段行动清单

1. 在 `EcatController::Initialize()` 成功后，通过 SDO 读取从站关键对象字典验证通信。
2. 将 SDO 验证结果输出到日志，并考虑封装成 `ValidateSdoCommunication()` 之类的私有方法。
3. 研究 RA8 的 PDO 映射对象（如 `0x1C00` 同步管理器、`0x1600`/`0x1A00` PDO 映射），为动态读取做准备。
4. 后续 PDO 阶段替换 `SlaveConfigFactory` 的硬编码伺服映射。
5. 验证 `CheckSlaveStates()` 在 OP 阶段检测到掉线后进入 `kError` 的行为。
6. 状态机重构编译验证通过（已达成），后续上电验证 PREOP 初始化仍然正常。
