## 目标

在主站状态机框架已完成、Activity 框架已扩展的基础上，进一步把 `EcatApplication` 改造为**显式 switch-case 主循环**，让状态机更直观，并为周期性任务提供统一入口。

**当前阶段仍不进入 OP。**

---

## 当前重点：switch-case 主循环

### 1. 新增 `EcatApplication::Run()`

```cpp
while (!shutdown_requested_) {
    bool has_command = ReadCommand(command, 100);

    switch (controller_.GetState()) {
    case ControllerState::kInitDone:    HandleInitDoneState(...);  break;
    case ControllerState::kScanned:     HandleScannedState(...);   break;
    case ControllerState::kPreOp:       HandlePreOpState(...);     break;
    case ControllerState::kOperational: HandleOperationalState(...); break;
    case ControllerState::kError:       HandleErrorState(...);     break;
    }
}
```

### 2. 各状态职责

| 状态 | 周期任务 | 允许的命令 |
|---|---|---|
| `kInitDone` | 无 | `scan`, `stop`, `help`, `exit` |
| `kScanned` | 无 | `config`, `stop`, `help`, `exit` |
| `kPreOp` | `CheckSlaveStates()` | `diagnose`, `param`, `inspect`, `start`, `stop`, `help`, `exit` |
| `kOperational` | `RunOneCycle()` + `CheckSlaveStates()` | `stop`, `exit` |
| `kError` | 无 | `stop`, `exit` |

### 3. 退出机制

- `exit` / `quit` / Ctrl+D：请求退出主循环。
- `Ctrl+C` / `SIGTERM`：通过信号处理函数调用 `RequestShutdown()`。

---

## 验收标准

1. `EcatApplication::Run()` 使用 switch-case 直接根据 `ControllerState` 分发。
2. `kPreOp` 状态下周期性调用 `CheckSlaveStates()`。
3. `kOperational` 状态下周期性调用 `RunOneCycle()` + `CheckSlaveStates()`（当前为框架占位，周期精度后续优化）。
4. 命令按状态过滤，非法命令给出明确错误提示。
5. `Ctrl+C` 和 `exit` 都能正常退出。
6. 编译通过，`ecat_scan` 仍能让从站进入 PREOP 并执行 SDO 诊断。
7. 不进入 SAFEOP/OP，不启动独立 PDO 周期线程。
