# 当前任务

## 目标

在主站状态机框架已完成、从站能稳定进入 **PREOP** 的基础上，验证 **SDO 通信**，并为后续 PDO 阶段实现动态 PDO 映射读取打好基础。

**当前阶段仍不进入 OP。**

---

## 当前重点：SDO 通信验证

### 1. 通过 SDO 读取关键对象字典

初始化到 PREOP 后，通过 SDO 读取从站以下对象验证通信：

| Index:Sub | 含义 | 预期值/作用 |
|---|---|---|
| `0x1000:00` | Device type | 确认从站设备类型 |
| `0x1018:01` | Vendor ID | 应为 `0x00000766` |
| `0x1018:02` | Product Code | 应为 `0x00002000` |
| `0x1018:03` | Revision Number | 确认固件版本 |

### 2. 封装 SDO 验证逻辑

- 在 `EcatController` 中增加私有方法（如 `ValidateSdoCommunication()`）。
- 对 `SlaveNodeManager` 中的每个节点调用 `SdoRead()`。
- 将结果打印到日志；失败时返回 false，让 `Initialize()` 进入错误状态。

### 3. 暴露更友好的 SDO 接口（可选）

评估是否在 `SlaveNode` 上增加模板/类型安全封装，例如：

```cpp
template <typename T>
bool SdoRead(uint16_t index, uint8_t subindex, T *value, int timeout_us = 10000);
```

当前阶段先保持原始 `void*` 接口，够用即可。

---

## 验收标准

1. `Initialize()` 到 PREOP 后，能通过 SDO 成功读取 `0x1000:00` 和 `0x1018` 子索引。
2. SDO 读取失败时，`Initialize()` 返回 false 并进入 `kError` 状态。
3. 日志输出清晰的 SDO 验证结果。
4. 编译通过，`ecat_scan` 仍能让从站进入 PREOP。
5. 不进入 SAFEOP/OP，不启动 PDO 周期线程。
