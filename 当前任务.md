# 当前任务

## 目标

在当前已实现"初始化到 PREOP"的基础上，进一步构建**健壮、可扩展的主站状态机框架**，为后续 PDO 阶段、多从站管理、错误恢复等复杂场景打好基础。

**当前阶段仍不涉及 PDO 通信。**

---

## 当前重点：重构状态机框架

### 1. 引入统一状态转换入口

- 实现 `TransitionTo(ControllerState target)`
- 所有状态转换（Initialize / StartOperation / Stop）统一走该入口
- 集中处理：前置条件检查、状态转换执行、失败回滚、日志记录

### 2. 定义状态转换表

- 明确每个状态允许转换到哪些目标状态
- 防止非法状态跳转
- 例如：`kPreOp` 只能转到 `kPdoConfigured` 或 `kUninitialized`

### 3. 拆分状态转换实现

- `DoInit()`：绑定网卡
- `DoScan()`：扫描从站
- `DoPreOp()`：进入 PREOP 并确认
- `DoPdoConfigure()` / `DoSafeOp()` / `DoDcConfigure()` / `DoOperational()`：后续 PDO 阶段预留
- `DoShutdown()`：回滚到 INIT 并清理资源
- `DoError()`：进入错误状态

### 4. 清理 `operational_` 标志

- 去掉 `operational_` 成员变量
- 用 `state_ == ControllerState::kOperational` 判断

### 5. 实现 `kError` 状态

- 增加 `EnterErrorState(const std::string &reason)`
- 状态异常时进入 Error 状态

### 6. 优化 `Shutdown()` 回滚路径

- 根据当前 `state_` 决定回滚步骤
- PREOP 时直接回 INIT，不请求 SAFEOP

### 7. 简化 `Initialize()` 日志输出

- 避免重复打印完整 SlaveInfo
- 只保留关键状态转换点的日志

---

## 验收标准

1. `Initialize()` 仍只到 `kPreOp` 即完成。
2. `Stop()` 能根据当前状态正确回滚到 INIT。
3. 非法状态转换被拦截并打印错误。
4. 状态转换失败时进入 `kError` 或正确回滚，不留在中间状态。
5. 编译通过，`ecat_scan` 仍能让从站进入 PREOP。
6. 代码结构清晰，新增状态（如 `kPdoConfigured`）时只需扩展对应 `DoXxx()` 和转换表。
