# 主站正常流程

1. **扫描从站**
   - 主站识别总线上的所有从站。
   - 读取每个从站的 SII/EEPROM 信息，并据此配置邮箱对应的 Sync Manager 0/1。

2. **进入 PREOP**
   - 所有从站进入 PREOP 状态，mailbox 建立。
   - 主站为每个从站建立管理对象，SDO 通信就绪。

3. **SDO 维护**
   SDO（Service Data Object）基于 CoE 邮箱，用于非周期数据交换。PREOP 阶段通过 SDO 完成以下典型维护工作：

   - **从站身份确认**
     - 读取对象字典 `0x1018`，获取 vendor_id、product_id、revision_number、serial_number，确认从站身份和固件版本。

   - **状态与故障诊断**
     - 读取从站当前 EtherCAT 状态、AL status code、错误码（如 `0x603F`），判断是否存在通信或应用层故障。

   - **参数读写**
     - 读取或修改对象字典中的配置参数，例如限位、控制模式、PDO 映射索引、电机相关参数等。
     - 写入后通常回读校验，确保参数生效。

   - **PDO 映射查看与配置（可选）**
     - 读取 `0x1C00`（Sync Manager 通信类型）、`0x1C12`/`0x1C13`（PDO 分配）、`0x1600`/`0x1A00`（PDO 映射），为后续进入 OP 做准备。

   - **固件与版本巡检**
     - 读取固件版本 `0x100A`、硬件版本 `0x1009` 等，用于版本管理和兼容性检查。

4. **进入 OP（可选）**
   - 配置 PDO 映射、SafeOp、DC 同步。
   - 所有从站进入 OPERATIONAL，开始周期性 PDO 数据交换。

5. **周期运行与监控**
   - 每个周期执行 PDO 收发。
   - 持续监控从站状态，异常时停机并回滚到安全状态。
