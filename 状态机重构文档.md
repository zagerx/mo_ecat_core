# 状态机重构文档

## 1. 文档目的

本文档描述 `EcatController` 主站状态机的具体实现规范，用于指导当前及后续基于该状态机的开发工作。

状态机采用“**主站生命周期状态 + 独立维护活动**”的二维设计。本文档只描述**主站生命周期状态机**这一部分，维护活动（`EcatActivity`）的设计见《主站状态设计方案.md》。

---

## 2. 状态定义

```cpp
enum class ControllerState {
    kUninitialized,   // 初始状态
    kInitDone,        // SOEM 初始化完成，网卡已绑定
    kScanned,         // 从站扫描完成
    kPreOp,           // 所有从站进入 PREOP，SlaveNode 占位对象已创建
    kPdoConfigured,   // 从站 Configure() 完成（PDO 阶段）
    kSafeOp,          // 所有从站进入 SAFEOP，IOmap 已建立（PDO 阶段）
    kDcConfigured,    // DC 同步已配置（PDO 阶段）
    kOperational,     // 所有从站进入 OPERATIONAL（PDO 阶段）
    kError            // 异常状态，只能 Stop 回到 Uninitialized
};
```

当前阶段（PREOP + SDO）只使用到 `kPreOp`。后续 PDO 阶段才会进入 `kPdoConfigured` 及之后状态。

## 3. 状态转换规则

### 3.1 允许的直接转换表

```cpp
static const std::map<ControllerState, std::vector<ControllerState>> kAllowedTransitions = {
    {kUninitialized,  {kInitDone}},
    {kInitDone,       {kScanned,        kUninitialized}},
    {kScanned,        {kPreOp,          kUninitialized}},
    {kPreOp,          {kPdoConfigured,  kUninitialized}},
    {kPdoConfigured,  {kSafeOp,         kPreOp, kUninitialized}},
    {kSafeOp,         {kDcConfigured,   kPreOp, kUninitialized}},
    {kDcConfigured,   {kOperational,    kSafeOp, kUninitialized}},
    {kOperational,    {kSafeOp,         kUninitialized}},
    {kError,          {kUninitialized}}
};
```

### 3.2 自动顺序前进

如果目标状态在当前状态之后，`TransitionTo()` 会按枚举顺序自动逐步前进，每步调用对应的 `DoXxx()`。

例如 `Initialize()` 调用 `TransitionTo(kPreOp)`：

```text
Uninitialized → InitDone → Scanned → PreOp
```

例如 `StartOperation()` 调用 `TransitionTo(kOperational)`：

```text
PreOp → PdoConfigured → SafeOp → DcConfigured → Operational
```

### 3.3 后退与特殊转换

后退转换（如 `kOperational → kUninitialized`）必须通过 `kAllowedTransitions` 表检查。`Stop()` 统一调用 `TransitionTo(kUninitialized)`，由 `DoShutdown()` 根据当前状态逐级回滚。

### 3.4 非法转换

- 从 `kError` 只能 `Stop()` 回到 `kUninitialized`。
- 从 `kUninitialized` 不能直接跳到 `kOperational`。
- 非法转换会打印错误并返回 false。

## 4. 状态转换入口

### 4.1 统一入口

```cpp
bool TransitionTo(ControllerState target);
```

所有状态转换统一走该入口，集中处理：

- 前置条件检查
- 转换合法性检查
- 自动顺序前进或后退转换执行
- 失败回滚或进入 `kError`
- 日志记录

### 4.2 公共接口

```cpp
// 初始化到 kPreOp
bool Initialize(const EcMasterConfig &config);

// 进入 kOperational（PDO 阶段使用；当前 main 不调用）
bool StartOperation();

// 安全停止，回到 kUninitialized
void Stop();
```

实现：

```cpp
bool EcatController::Initialize(const EcMasterConfig &config)
{
    if (state_ != ControllerState::kUninitialized) {
        LOG_WARN << "Already initialized, state=" << StateToString(state_);
        return false;
    }

    config_ = config;
    return TransitionTo(ControllerState::kPreOp);
}

bool EcatController::StartOperation()
{
    if (state_ != ControllerState::kPreOp) {
        LOG_WARN << "StartOperation called in invalid state " << StateToString(state_);
        return false;
    }

    return TransitionTo(ControllerState::kOperational);
}

void EcatController::Stop()
{
    if (state_ == ControllerState::kUninitialized) {
        return;
    }

    TransitionTo(ControllerState::kUninitialized);
}
```

> `StartOperation()` 当前语义：**从 `kPreOp` 开始执行完整 PDO 启动流程**。后续如需支持从中间状态继续进入 OP，可放宽前置条件。

## 5. 各状态转换步骤

### 5.1 DoInit()

- 前置状态：`kUninitialized`
- 操作：`master_.Initialize(config_)`
- 成功：`state_ = kInitDone`
- 失败：保持 `kUninitialized`

### 5.2 DoScan()

- 前置状态：`kInitDone`
- 操作：`master_.ScanSlaves()`，保存 `slave_infos_`
- 成功：`state_ = kScanned`，打印 SlaveInfo
- 失败：调用 `DoShutdown()` 回滚

### 5.3 DoPreOp()

- 前置状态：`kScanned`
- 操作：
  - `master_.RequestPreOpState()`
  - `master_.CheckAllSlavesInState(EC_STATE_PRE_OP)`
  - `slave_infos_ = RefreshSlaveInfos()`
  - `node_manager_.Initialize(master_, slave_infos_)`
- 成功：`state_ = kPreOp`
- 失败：调用 `DoShutdown()` 回滚

### 5.4 DoPdoConfigure()

- 前置状态：`kPreOp`
- 操作：`node_manager_.ConfigureAll()`
- 成功：`state_ = kPdoConfigured`
- 失败：`EnterErrorState()`

### 5.5 DoSafeOp()

- 前置状态：`kPdoConfigured`
- 操作：
  - `master_.ConfigureProcessData()`
  - `master_.RequestSafeOpState()`
  - `master_.CheckAllSlavesInState(EC_STATE_SAFE_OP)`
- 成功：`state_ = kSafeOp`
- 失败：`EnterErrorState()`

### 5.6 DoDcConfigure()

- 前置状态：`kSafeOp`
- 操作：`master_.ConfigureDc()`
- 成功：`state_ = kDcConfigured`
- 失败：`EnterErrorState()`

> DC 失败策略需后续明确：
> - `use_dc == false`：跳过，不视为失败；
> - `use_dc == true`：配置失败进入 `kError`；
> - 部分从站不支持 DC 时是否允许跳过，视实际需求决定。

### 5.7 DoOperational()

- 前置状态：`kDcConfigured`
- 操作：
  - `master_.RequestOperationalState()`
  - `master_.CheckAllSlavesInState(EC_STATE_OPERATIONAL)`
- 成功：`state_ = kOperational`
- 失败：`EnterErrorState()`

### 5.8 DoShutdown()

- 作用：根据当前状态逐级回滚，最终关闭 SOEM
- 回滚路径：

```text
Operational / DcConfigured -> SAFE_OP
SafeOp / Operational / DcConfigured -> PRE_OP
PreOp / PdoConfigured / SafeOp / Operational / DcConfigured -> INIT
InitDone / Scanned -> 直接 Close
Uninitialized -> 无操作
```

- 成功：`state_ = kUninitialized`

## 6. 错误处理

### 6.1 EnterErrorState()

```cpp
void EnterErrorState(const std::string &reason)
{
    LOG_ERROR << "Entering error state: " << reason;
    state_ = ControllerState::kError;
}
```

### 6.2 初始化阶段失败

`DoInit()`、`DoScan()`、`DoPreOp()` 失败时调用 `DoShutdown()`，最终回到 `kUninitialized`，不进入 `kError`。

### 6.3 PDO 阶段失败

`DoPdoConfigure()` 及之后阶段失败时进入 `kError`。此时只能调用 `Stop()` 回到 `kUninitialized`。

### 6.4 运行中掉线

`CheckSlaveStates()` 在 `kOperational` 时检测从站是否掉出 OP：

```cpp
void EcatController::CheckSlaveStates()
{
    master_.CheckSlaveStates();

    if (state_ == ControllerState::kOperational) {
        if (!master_.CheckAllSlavesInState(EC_STATE_OPERATIONAL)) {
            EnterErrorState("Slave dropped out of OPERATIONAL");
        }
    }
}
```

> 后续增强：PREOP / SAFEOP 阶段也应检测对应状态，但当前阶段不作为优先项。

## 7. 周期任务与状态检查

### 7.1 RunOneCycle()

只在 `kOperational` 时执行 PDO 周期：

```cpp
void EcatController::RunOneCycle()
{
    if (state_ != ControllerState::kOperational) {
        LOG_WARN << "RunOneCycle ignored: not operational";
        return;
    }

    node_manager_.UpdateAllOutputs();
    master_.RunOneCycle();
    node_manager_.UpdateAllInputs();
}
```

### 7.2 CheckSlaveStates()

由上层或后续监控线程调用。当前只在 OP 阶段检测掉线。

## 8. 状态查询接口

```cpp
bool IsInitialized() const
{
    return state_ != ControllerState::kUninitialized;
}

bool IsOperational() const
{
    return state_ == ControllerState::kOperational;
}
```

> `operational_` 和 `initialized_` 成员已移除，统一基于 `state_` 判断。

## 9. 使用示例

### 9.1 当前 PREOP 阶段用法

```cpp
mo_ecat::EcMasterConfig config;
config.ifname = "enp0s31f6";
config.cycle_time_us = 1000;
config.use_dc = true;

mo_ecat::EcatController controller;
if (!controller.Initialize(config)) {
    return 1;
}

// controller.GetState() == kPreOp
// 可通过 GetSlaveNodeManager() 访问占位节点，进行 SDO 通信

controller.Stop();
```

### 9.2 后续 PDO 阶段用法

```cpp
if (!controller.Initialize(config)) return 1;

if (!controller.StartOperation()) return 1;
// controller.GetState() == kOperational

while (running) {
    controller.RunOneCycle();
    controller.CheckSlaveStates();
    if (!controller.IsOperational()) {
        // 进入 kError，需要 Stop() 后恢复
        break;
    }
}

controller.Stop();
```

## 10. 与 Activity 方案的关系

- 本文档描述的线性状态机负责主站正常运行生命周期。
- `EcatActivity` 负责维护操作（FOE 升级、SDO 诊断、单站复位等），不改变 `ControllerState`。
- `EcatController::ExecuteActivity()` 在活动开始前检查 `CanStart(state_)`，确保活动在允许的主站状态下执行。
- 高风险 Activity（如 FOE 升级）第一阶段建议通过 `TransitionTo()` 先把主站降到 SAFEOP/PREOP，再执行 Activity。

---

## 11. 注意事项

1. **不要直接修改 `state_`**：所有状态转换必须通过 `TransitionTo()`。
2. **不要重复初始化**：`Initialize()` 要求 `state_ == kUninitialized`。
3. **错误状态只能 Stop**：`kError` 状态下只有 `Stop()` 能回到 `kUninitialized`。
4. **前进路径也建议查表**：虽然当前按枚举自动前进，但建议每步调用 `IsTransitionAllowed(state_, next)`。
5. **日志**：状态转换关键节点打印日志；扫描后打印一次完整 SlaveInfo，进入 PREOP 后只打印 summary。
