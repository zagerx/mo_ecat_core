# 人形机器人实时控制接口规范

## 1. 文档定位

本文档细化《人形机器人 EtherCAT 主站功能规划》中的实时上层接口部分，用于定义运动控制器与 EtherCAT 主站之间的周期命令、反馈、时序和异常处理边界。

主站不负责轨迹规划和动力学计算，但必须保证上层下发的周期目标能够确定性地进入 PDO 输出，并将驱动器反馈稳定返回给上层。

## 2. 接口原则

1. **周期语义明确**：每帧命令和反馈都必须携带周期序号或时间戳。
2. **实时线程不阻塞**：主站周期线程不得等待非实时接口、日志、文件 IO 或 RPC。
3. **命令单调可判定**：过期、重复、跳变过大的命令必须可检测。
4. **反馈完整可追踪**：反馈必须包含通信状态、伺服状态、故障状态和周期统计。
5. **安全策略内建**：命令超时、异常命令、接口断开必须触发明确保护策略。

## 3. 命令帧

| 字段 | 必需 | 说明 |
|------|------|------|
| sequence | 是 | 上层周期序号，必须单调递增 |
| timestamp_ns | 是 | 上层生成命令的时间戳 |
| control_mode | 是 | CSP / CSV / CST / Homing / Disable 等模式 |
| axis_count | 是 | 命令包含的轴数量 |
| axis_id | 是 | 轴唯一标识，建议与配置中的关节名绑定 |
| target_position | CSP 必需 | 目标位置，工程单位 |
| target_velocity | CSV 或前馈可用 | 目标速度，工程单位 |
| target_torque | CST 或前馈可用 | 目标力矩，工程单位 |
| enable_mask | 是 | 哪些轴允许接收本帧命令 |
| safety_flags | 是 | 上层请求的保持、减速、停止等安全标志 |

## 4. 反馈帧

| 字段 | 必需 | 说明 |
|------|------|------|
| sequence | 是 | 主站反馈周期序号 |
| command_sequence | 是 | 本周期实际采用的命令序号 |
| timestamp_ns | 是 | 主站生成反馈的时间戳 |
| master_state | 是 | 主站生命周期状态 |
| bus_state | 是 | 总线状态、WKC、DC 状态 |
| axis_state | 是 | 每个轴的 CiA402 状态、模式、故障码 |
| actual_position | 是 | 实际位置 |
| actual_velocity | 是 | 实际速度 |
| actual_torque | 是 | 实际力矩或电流换算值 |
| status_word | 是 | `0x6041` |
| error_code | 是 | `0x603F` 或厂商错误码 |
| cycle_time_ns | 是 | 本周期耗时 |
| jitter_ns | 是 | 本周期 jitter |

## 5. 时序要求

| 项目 | 要求 |
|------|------|
| 命令更新 | 每个 EtherCAT 周期最多采用一帧上层命令 |
| 过期命令 | 发现 sequence 未更新或 timestamp 超时后触发命令超时策略 |
| 重复命令 | 允许短时间 Hold，但必须计数并上报 |
| 未来命令 | timestamp 明显超前时拒绝使用 |
| 反馈发布 | 每个周期发布一次反馈或更新共享缓冲 |
| 接口抖动 | 不得直接扩大 EtherCAT 周期 jitter |

## 6. 命令保护

主站写入 PDO 前必须执行：

- 轴 ID 与配置匹配检查。
- 控制模式合法性检查。
- 位置、速度、力矩限幅。
- 目标突变检查。
- 软限位检查。
- 轴组状态检查。
- 命令时间戳或序号检查。
- 急停、STO、硬限位状态检查。

## 7. 推荐实现形态

| 方式 | 适用场景 | 注意事项 |
|------|----------|----------|
| 共享内存 + 双缓冲 | 同机高实时控制 | 需要版本号、序号和内存屏障 |
| Lock-free Ring Buffer | 高频命令和反馈 | 需要处理覆盖和读写速率不匹配 |
| DDS / ROS 2 | 调试、分布式系统 | 不建议直接阻塞实时周期线程 |
| RPC / Socket | 维护命令和诊断 | 不作为强实时命令通道 |

## 8. 验收要求

实时接口至少需要验证：

- 正常 1ms 周期命令和反馈连续运行。
- 命令 sequence 丢失、重复、乱序、超时。
- 目标位置、速度、力矩越限。
- 上层进程退出或接口断开。
- 高负载下 EtherCAT 周期 jitter 不被接口显著放大。
- 故障时反馈帧能够携带完整故障上下文。
