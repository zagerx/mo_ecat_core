## 2. 目标场景与边界

### 2.1 目标场景

主站用于人形机器人全身多轴控制，典型对象包括腿部、腰部、手臂、颈部、手部等伺服驱动器，以及必要的 IO、安全、传感器类 EtherCAT 从站。

重点能力：

- 20~50+ 轴级别的周期性伺服控制。
- 1ms 默认控制周期，具备向 500us 周期演进的工程路径。
- 多轴 DC 同步与稳定 Working Counter 监测。
- CiA402 伺服状态机、CSP 为主的周期位置控制，兼顾 CSV/CST 扩展。
- 面向整机安全的分级停机、分组恢复、故障诊断和数据追溯。

### 2.2 职责边界

主站负责：

- EtherCAT 通信、状态机、PDO/SDO、DC 同步、拓扑管理。
- 从站抽象、轴组管理、伺服状态机、故障处理。
- 向上层控制器提供确定性的实时命令/反馈接口。

主站不负责：

- 人形机器人运动学、动力学、步态规划、轨迹规划。
- 高层行为决策、任务调度、感知融合。
- 功能安全认证本身；但需要为外部安全链路预留接口。

### 2.3 分级定义

| 分级 | 含义 | 验收要求 |
|------|------|----------|
| M（Must） | 人形机器人主站的基础必需能力 | 第一阶段或核心版本必须具备 |
| S（Should） | 显著提升调试、稳定性、量产维护的能力 | 建议随核心版本迭代实现 |
| O（Optional） | 高阶产品、特殊硬件或安全认证场景需要 | 按产品定位和硬件条件实施 |

## 3. 设计原则

1. **状态机驱动**：主站、从站、伺服轴、轴组均有明确状态，所有操作必须受状态约束。
2. **确定性优先**：周期 PDO 收发、命令更新、反馈读取必须满足固定周期预算。
3. **安全优先**：通信异常、伺服异常、限位异常、上层命令超时必须进入可控降级路径。
4. **配置与代码分离**：从站身份、PDO 映射、SDO 初始化、缩放、限位、故障策略通过配置描述。
5. **分层解耦**：通信层、设备抽象层、伺服语义层、实时接口层、诊断工具层职责清晰。
6. **可观测性内建**：状态迁移、周期时间、WKC、DC 偏差、SDO/PDO 错误、伺服故障均可记录与追溯。
7. **面向整机维护**：支持轴组级启动、停止、诊断、恢复和数据记录，适配人形机器人现场调试。

## 4. 总体能力模型

人形机器人 EtherCAT 主站应具备以下核心能力域：

| 能力域 | 目标 |
|--------|------|
| 通信与总线管理 | 建立稳定 EtherCAT 网络，管理拓扑、状态和链路异常 |
| 主站生命周期 | 控制初始化、扫描、维护、运行、错误、停止等阶段 |
| PDO 过程数据 | 构建 IOmap，周期收发控制命令与反馈数据 |
| SDO 服务数据 | 支持参数配置、诊断、对象字典访问和启动配置 |
| DC 同步 | 支撑多轴同步运动和确定性周期控制 |
| CiA402 伺服控制 | 管理控制字/状态字、运行模式、故障复位和反馈解析 |
| 人形机器人轴组 | 支持腿、臂、腰、头等分组控制和分组诊断 |
| 实时上层接口 | 与运动控制器交换带周期语义的命令和反馈 |
| 安全与故障处理 | 分级故障、分级停机、恢复流程和安全接口 |
| 诊断与数据记录 | 支持现场调试、问题复现、数据回放和健康监测 |
| 配置与工具 | 支持配置文件、命令行、离线校验、ESI/Profile 管理 |
| 测试与验收 | 为每项能力定义可执行的验证标准 |

## 5. 分阶段路线图

### 5.1 Phase 1：通信与维护基础

目标是建立稳定的 EtherCAT 主站基础能力，使系统能够完成从网卡绑定到 PREOP/维护状态的可靠流程。

必须具备：

- 网卡初始化、从站扫描、身份读取、拓扑识别。
- INIT / PREOP / SAFEOP / OP / BOOT 状态请求与确认。
- SDO 读写、批量 SDO 配置、回读校验。
- 从站列表、从站身份、AL status code、错误码诊断。
- 主站生命周期状态机和错误状态。
- 基础日志、命令行诊断、配置文件加载。

### 5.2 Phase 2：周期 PDO 与单/多轴伺服运行

目标是让主站具备稳定 OP 运行能力，支持多轴周期控制。

必须具备：

- PDO 映射读取、配置、校验与 IOmap 构建。
- 周期 PDO 收发、WKC 检查、输入输出解析与打包。
- DC 检测、配置、同步偏差监控和主站周期对齐。
- CiA402 状态机、CSP 模式、故障复位、Quick Stop。
- 周期抖动、周期耗时、丢包、WKC 异常日志。
- 单轴、多轴基础验收测试。

### 5.3 Phase 3：人形机器人整机能力

目标是支撑人形机器人多轴整机调试与长期运行。

必须具备：

- 腿、臂、腰、头、手等轴组抽象。
- 轴组级启动、使能、停止、诊断和恢复。
- 上层实时控制接口，支持命令时间戳、周期序号、命令超时保护。
- 抱闸、限位、跟随误差、编码器异常、过温、过流等关节级安全策略。
- 全身 PDO 数据记录、故障前后窗口快照、数据导出。
- 20~50+ 轴压力测试、长时间运行测试和故障注入测试。

### 5.4 Phase 4：工程化与高级能力

根据产品定位选择：

- 在线重配置、在线 PDO 重映射。
- 热连接、从站更换识别、产线维护工具。
- 网络冗余、链路质量监控。
- FSoE、安全 PLC、STO 双通道接口。
- ESI 自动解析、Profile 自动生成。
- 离线仿真、数据回放、HIL 测试平台。

## 6. 功能清单

### 6.1 通信与总线管理

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|
| 网卡初始化与绑定 | M | 选择指定网卡，初始化 SOEM 或自研驱动 | 指定网卡不存在时明确报错，存在时可进入扫描阶段 |
| 从站扫描与枚举 | M | 读取 Vendor / Product / Revision / Serial，构建从站列表 | 扫描结果与实际链路顺序一致 |
| 拓扑识别 | M | 识别物理顺序、端口状态、分支/环网结构 | 能输出从站顺序和端口连接状态 |
| 配置地址分配 | M | 为每个从站分配 configured station address | 每个从站地址唯一且可用于后续访问 |
| 从站状态机控制 | M | 支持 INIT / PREOP / SAFEOP / OP / BOOT 请求与确认 | 状态切换失败时能读取 AL status code |
| 总线状态监控 | M | 周期性读取从站实际状态与 AL status code | 任意从站状态异常可进入故障处理 |
| 链路异常检测 | S | 检测掉站、链路断开、端口异常 | 掉线在配置周期数内被发现 |
| 网络冗余 | O | 支持线缆冗余或环网冗余 | 断开单段链路后通信可恢复 |
| 热插拔 / 热连接 | O | 运行中识别新从站或掉线恢复 | 插拔事件不破坏未受影响轴组的诊断能力 |

### 6.2 主站生命周期

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|
| 主站状态机 | M | 定义稳定状态与合法迁移，禁止非法跳转 | 非法命令被拒绝并记录原因 |
| 分阶段启动 | M | 初始化、扫描、维护、启动运行分阶段执行 | 每阶段失败可定位原因 |
| 维护状态 | M | 在非 OP 状态下执行 SDO 维护、参数配置和诊断 | 可反复执行诊断命令且不污染运行状态 |
| 运行状态 | M | OP 后执行周期 PDO 与状态监控 | 周期任务只在合法状态运行 |
| 安全停止 | M | 从任意状态进入可控停止流程并释放资源 | Stop 后可重新初始化 |
| 错误状态 | M | 不可恢复错误进入 Error，只允许 Stop/Reset/诊断 | Error 保留故障快照 |
| 在线重配置 | S | 从运行回到维护，修改参数后再启动 | 重配置失败不遗留半配置运行状态 |

### 6.3 PDO 过程数据

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|
| PDO 映射读取 | M | 读取 `0x1C12` / `0x1C13` 与 `0x1600~` / `0x1A00~` 映射 | 实际映射可被完整打印 |
| PDO 映射配置 | M | 通过 SDO 写 PDO 分配与映射对象 | 写入后回读一致 |
| PDO 映射校验 | M | 校验从站实际映射与主站 Profile 预期一致 | 不一致时禁止进入 OP |
| IOmap 构建 | M | 计算每个从站 outputs/inputs 偏移，配置 FMMU/SM | 每个对象偏移、位宽、方向可查询 |

| PDO 数据打包 | M | 将上层命令编码为 IOmap 输出 | 输出字节序、位宽和符号处理正确 |
| PDO 数据记录 | M | 记录关键控制命令、反馈、状态字和 WKC | 故障前后窗口可导出分析 |
| PDO 在线重映射 | S | 不重启进程修改 PDO 映射并重新进入 OP | 失败时回到维护或错误状态 |

### 6.9 安全与故障处理

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|
| 安全状态机 | M | 定义 Normal / Warning / RecoverableFault / FatalFault / Emergency | 不同等级对应不同处理策略 |
| 通信故障处理 | M | 掉站、WKC 异常、状态丢失、SDO 超时处理 | 触发后可定位从站和故障类型 |
| 伺服故障处理 | M | 跟随误差、过流、过温、编码器异常、限位异常 | 故障轴进入明确状态 |
| Quick Stop 策略 | M | 首选软件 Quick Stop，失败时进入更高等级停机 | 可配置减速度和超时 |
| Disable Voltage 策略 | S | 对特定故障执行失能 | 失能前后状态可追踪 |
| Fault Reset 流程 | M | 故障复位前检查原因、状态和用户确认条件 | 未清除故障不允许重新使能 |
| 外部急停输入 | M | 接入 E-stop 或安全 IO 状态 | 急停触发优先级高于普通命令 |
| STO 接口 | S | 与外部 STO 继电器或安全板交互 | 主站可观测 STO 状态 |
| FSoE 支持 | O | Safety over EtherCAT | 需独立安全设计和认证路径 |
| 双通道安全监控 | O | 独立安全 PLC 或安全核监控主站行为 | 主站提供必要状态输出 |

详细故障分级和停机策略见：[人形机器人主站安全与故障策略](人形机器人主站安全与故障策略.md)。

### 6.10 诊断、日志与数据记录

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|
| 结构化日志 | M | 状态迁移、命令、SDO/PDO 错误、AL code、伺服错误 | 日志可按时间和轴号检索 |
| 日志级别动态调整 | S | 运行时切换 debug/info/warn/error | 不重启进程即可调整 |
| 从站快照 | M | Error 时记录所有从站状态、AL code、错误寄存器 | 故障后可导出快照 |
| 轴状态快照 | M | 记录控制字、状态字、模式、目标值、反馈值 | 可复盘故障瞬间状态 |
| SDO 诊断 | S | 读取 `0x1000` / `0x1018` / `0x1008~0x100A` 等对象 | 可确认身份和版本 |
| PDO 映射诊断 | S | 显示 PDO 分配、映射、偏移、位宽 | 可定位映射错误 |
| 周期性能统计 | M | 记录周期耗时、jitter、WKC、DC 偏差 | 支持 p99/p999 统计 |
| 数据记录 | M | 记录关键 PDO、命令、反馈和故障窗口 | 可用于步态调试和故障复现 |
| 数据回放 | S | 离线回放命令和反馈，辅助复现问题 | 可驱动非硬件逻辑测试 |
| 健康监测 | S | 统计掉站次数、重试次数、温度、电流、错误频率 | 支持预测性维护 |

### 6.11 配置、Profile 与工具

| 功能 | 分级 | 说明 | 基础验收 |
|------|------|------|----------|

## 7. 非功能需求

| 需求 | 目标值 / 说明 |
|------|---------------|
| 默认周期 | 1ms |
| 扩展周期 | 500us，视硬件、轴数、驱动器能力和实时系统配置决定 |
| 轴数目标 | 第一目标 20~30 轴，扩展目标 50+ 轴 |
| 周期抖动 | 1ms 周期下 p99 jitter < 50us；更高要求按硬件实测确认 |
| DC 同步偏差 | 目标进入可配置同步窗口，超过阈值告警或停机 |
| 掉线检测 | 任意关键从站掉线在 3 个周期内发现并触发安全响应 |
| 命令超时 | 上层实时命令超时在配置周期数内触发保持、减速或停机 |
| 启动时间 | 典型配置从 scan 到 OP 小于 5s；大轴数按配置规模分级验收 |
| 连续运行 | 目标支持 8h、24h、72h 分级稳定性测试 |
| 日志持久化 | 日志按日期/大小轮转，保留策略可配置 |
| 数据记录 | 故障前后窗口记录长度可配置，支持导出 |
| 配置热加载 | 非实时、非运行关键配置可热加载；影响 PDO/DC/安全策略的配置必须重新进入维护或启动流程 |

## 8. 测试与验收策略

### 8.1 测试分层

| 层级 | 测试对象 | 示例 |
|------|----------|------|
| 单元测试 | 状态机、配置解析、PDO offset、CiA402 状态解析 | 无硬件运行 |
| 集成测试 | SOEM 封装、SDO/PDO、DC 配置、从站状态切换 | 连接真实或仿真从站 |
| 整机测试 | 多轴 OP、轴组启动、故障停机、数据记录 | 人形机器人或台架 |
| 压力测试 | 轴数、周期、长时间运行、异常注入 | 20~50+ 轴目标 |
| 回归测试 | 已发现问题和现场故障 | 每次核心改动后执行 |

### 8.2 必要验收项

M 级能力必须至少具备以下验收材料：

- 测试步骤。
- 期望结果。
- 失败判定。
- 日志或数据记录样例。
- 对应配置样例。

### 8.3 故障注入

必须覆盖：

- 从站掉线。
- WKC 异常。
- DC 同步丢失或偏差超限。
- SDO 超时。
- 伺服 Fault。
- 跟随误差超限。
- 上层命令超时。
- E-stop 触发。
- 配置与实际从站不匹配。

## 9. 依赖与风险

| 方向 | 风险 | 建议 |
|------|------|------|
| SOEM 能力边界 | 在线重映射、热连接、冗余、安全能力需要额外封装或验证 | 先实现核心链路，再评估高级能力 |
| 实时系统 | 普通 Linux 难以保证高轴数低 jitter | 评估 PREEMPT_RT、CPU isolation、网卡中断亲和性 |
| 驱动器差异 | 不同厂商 CiA402、PDO、错误码存在差异 | 引入 DeviceProfile 和厂商适配层 |
| 安全认证 | FSoE/STO 涉及独立安全设计 | 主站只做接口与状态观测，认证另行规划 |
| 数据量 | 50+ 轴高频记录数据量大 | 采用环形缓冲、分级采样、故障窗口记录 |
| 配置复杂度 | 轴、轴组、Profile、ESI、固件版本容易不一致 | 配置必须可校验、可版本化、可追溯 |
