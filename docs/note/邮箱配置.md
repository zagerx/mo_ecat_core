# 邮箱配置分析

## 1. 邮箱概述

EtherCAT 邮箱（Mailbox）用于主站与从站之间的**非周期性数据通信**，典型应用包括：

- SDO（Service Data Object）读写
- FoE（File over EtherCAT）
- EoE（Ethernet over EtherCAT）
- CoE（CANopen over EtherCAT）

每个支持邮箱通信的从站至少需要两个 SyncManager：

| SyncManager | 方向 | 名称 | 典型地址 | 长度 |
|-------------|------|------|----------|------|
| SM0 | 主站写 → 从站读 | Mailbox Out | `0x1000` | 128 bytes |
| SM1 | 从站写 → 主站读 | Mailbox In | `0x1080` | 128 bytes |

> 本项目从站的邮箱配置：`SM0 @ 0x1000`，`SM1 @ 0x1080`，长度均为 128 bytes。

---

## 2. 邮箱配置数据来源

邮箱的物理地址和长度来自从站 **EEPROM / ESI 配置**。

**ESI XML 文件** `temp/RA8 EtherCAT.xml`：

```xml
<Mailbox DataLinkLayer="true">
    <Send Start="0x1000" Length="128" />
    <Recv Start="0x1080" Length="128" />
</Mailbox>
```

主站在初始化阶段先通过 EEPROM 接口读取这些信息，然后再写入 SM 寄存器。

---

## 3. EEPROM 读取邮箱地址 —— Frame 96/106

### Frame 96：读取 SM0 邮箱地址

```text
No. 96 0.015999527  01:01:01:01:01:01 → Broadcast  ECAT 60
'FPRD': Len: 4, Adp 0x1001, Ado 0x508, Wc 1
```

展开：

```text
EtherCAT datagram: Cmd: 'FPRD' (4), Len: 4, Adp 0x1001, Ado 0x508, Cnt 1
    EEPROM Data 0 (0x508): 0x1000
    EEPROM Data 1 (0x50A): 0x0080
    Working Cnt: 1
```

解析：

| 字段 | 值 | 说明 |
|------|-----|------|
| Ado | `0x0508` | EEPROM 数据寄存器 |
| Data 0 | `0x1000` | SM0 物理起始地址 |
| Data 1 | `0x0080` | SM0 长度 = 128 bytes |

### Frame 106：读取 SM1 邮箱地址

```text
No. 106 0.017785551  03:01:01:01:01:01 → Broadcast  ECAT 60
'FPRD': Len: 4, Adp 0x1001, Ado 0x508, Wc 1
```

展开：

```text
EtherCAT datagram: Cmd: 'FPRD' (4), Len: 4, Adp 0x1001, Ado 0x508, Cnt 1
    EEPROM Data 0 (0x508): 0x1080
    EEPROM Data 1 (0x50A): 0x0080
    Working Cnt: 1
```

解析：

| 字段 | 值 | 说明 |
|------|-----|------|
| Data 0 | `0x1080` | SM1 物理起始地址 |
| Data 1 | `0x0080` | SM1 长度 = 128 bytes |

> 这两帧属于 `ecx_config_init()` 阶段的 EEPROM 读取过程。

---

## 4. SM 寄存器配置 —— Frame 267/268

读取 EEPROM 后，主站将邮箱配置写入 ESC 的 SM 寄存器区域。

### Frame 267：写入 SM0/SM1 配置

```text
No. 267 0.047320047  01:01:01:01:01:01 → Broadcast  ECAT 44
'FPWR': Len: 16, Adp 0x1001, Ado 0x800, Wc 0
```

展开：

```text
EtherCAT datagram: Cmd: 'FPWR' (5), Len: 16, Adp 0x1001, Ado 0x800, Cnt 0
    SyncManager: 0010800026000100
    SyncManager: 8010800022000100
    Working Cnt: 0
```

### Frame 268：从站响应

```text
No. 268 0.047599825  03:01:01:01:01:01 → Broadcast  ECAT 60
'FPWR': Len: 16, Adp 0x1001, Ado 0x800, Wc 1
```

`WKC=1` 表示成功写入 16 bytes（SM0 + SM1 配置）。

---

## 5. SM 配置数据解析

### SM0（Mailbox Out）

原始数据：`00 10 80 00 26 00 01 00`

小端解析：

| 字段 | 值 | 说明 |
|------|-----|------|
| StartAddr | `0x1000` | 邮箱输出缓冲区起始地址 |
| Length | `0x0080` | 128 bytes |
| Control | `0x26` | Mailbox 模式，主站写、从站读 |
| Status | `0x00` | 状态 |
| Activate | `0x01` | 使能 SM0 |
| PDI Control | `0x00` | |

### SM1（Mailbox In）

原始数据：`80 10 80 00 22 00 01 00`

小端解析：

| 字段 | 值 | 说明 |
|------|-----|------|
| StartAddr | `0x1080` | 邮箱输入缓冲区起始地址 |
| Length | `0x0080` | 128 bytes |
| Control | `0x22` | Mailbox 模式，主站读、从站写 |
| Status | `0x00` | 状态 |
| Activate | `0x01` | 使能 SM1 |
| PDI Control | `0x00` | |

### Control 字节对比

| SyncManager | Control | 二进制 | 含义 |
|-------------|---------|--------|------|
| SM0 Out | `0x26` | `0010 0110` | Mailbox 模式，ECAT 写，PDI 读 |
| SM1 In | `0x22` | `0010 0010` | Mailbox 模式，ECAT 读，PDI 写 |

Control 字节关键位：

- bit 0~1：操作模式
  - `10` = Mailbox 模式
- bit 2：方向
  - `1` = ECAT 写访问（主站写）
  - `0` = ECAT 读访问（主站读）

SM0 和 SM1 的唯一区别就是 bit 2，决定了主站是写还是读。

---

## 6. 邮箱初始化流程

```text
1. 主站读取 EEPROM
   ├── Frame 96:  读取 SM0 地址 0x1000，长度 128
   └── Frame 106: 读取 SM1 地址 0x1080，长度 128

2. 主站写入 SM 寄存器
   └── Frame 267: FPWR 0x0800 写入 SM0/SM1 配置

3. 从站使能邮箱
   └── Frame 268: 响应 WKC=1，邮箱配置生效

4. 后续通过 SM0/SM1 进行 SDO 通信
```

---

## 7. 邮箱与过程数据 SM 的区别

| 特性 | SM0/SM1（邮箱） | SM2/SM3（过程数据） |
|------|----------------|---------------------|
| 用途 | SDO、FoE、EoE 等非周期通信 | PDO 周期性通信 |
| 长度 | 通常 128 bytes | 由 PDO 大小决定 |
| 模式 | Mailbox | Buffered / Unbuffered |
| Control | `0x26` / `0x22` | `0x64` / `0x20` |
| 配置时机 | INIT / PRE-OP | PRE-OP / SAFE-OP |

---

## 8. 为什么邮箱配置很重要

如果 SM0/SM1 配置错误，主站无法通过 SDO 与从站通信，表现为：

- `ecx_SDOread()` / `ecx_SDOwrite()` 返回 `wkc=0`
- 无法读取对象字典
- 无法配置 PDO mapping

本项目中手动版本遇到的 `0x1C12/0x1C13 SDO read wkc=0` 就与邮箱状态有关。

## 9. ESC 相关寄存器

本节寄存器信息参考 **ETG.1000.4**（EtherCAT Data Link Layer Protocol Specification）和本项目从站 R-IN32 ESC 的 CMSIS 头文件。

### 9.1 ESC 寄存器总览（与本项目相关）

| 地址范围 | 名称 | 说明 |
|----------|------|------|
| `0x0000` ~ `0x000F` | ESC Type / Revision / RAM Size | ESC 基本信息 |
| `0x0100` ~ `0x0101` | DL Control | 数据链路层控制 |
| `0x0120` ~ `0x0121` | AL Control | 应用层状态控制（主站写） |
| `0x0130` ~ `0x0134` | AL Status / Code | 应用层状态反馈（从站写） |
| `0x0500` ~ `0x050F` | EEPROM Interface | EEPROM 读写接口 |
| `0x0600` ~ `0x06FF` | FMMU | FMMU 配置寄存器 |
| `0x0800` ~ `0x087F` | SyncManager | SM 配置寄存器 |

### 9.2 EEPROM 接口寄存器（0x0500 ~ 0x050F）

邮箱配置前，主站需要通过 EEPROM 接口读取 SII（Slave Information Interface）中的 SM 配置。

| 地址 | 名称 | 访问 | 说明 |
|------|------|------|------|
| `0x0500` | EEPROM Config | RW | EEPROM 配置/控制权 |
| `0x0502` | EEPROM Control/Status | RW | 读写命令与状态 |
| `0x0504` | EEPROM Address | RW | 读取/写入的 EEPROM 地址 |
| `0x0508` | EEPROM Data 0 | RW | 32-bit 数据低字 |
| `0x050A` | EEPROM Data 1 | RW | 32-bit 数据高字 |

本项目中的实际使用：

- **Frame 96/106**：主站读取 `0x0508` / `0x050A`，获得 EEPROM 中的 SM0/SM1 地址和长度。
- 读到的数据：
  - SM0: `0x1000`（地址），`0x0080`（长度 128）
  - SM1: `0x1080`（地址），`0x0080`（长度 128）

### 9.3 SyncManager 寄存器（0x0800 ~ 0x087F）

每个 SyncManager 通道占用 **8 bytes**，基址计算：

```text
SM0: 0x0800 + 0 * 8 = 0x0800
SM1: 0x0800 + 1 * 8 = 0x0808
SM2: 0x0800 + 2 * 8 = 0x0810
SM3: 0x0800 + 3 * 8 = 0x0818
...
```

单个 SM 寄存器结构：

| 偏移 | 字段 | 长度 | 说明 |
|------|------|------|------|
| +0 ~ +1 | Physical Start Address | 2 bytes | 缓冲区物理起始地址 |
| +2 ~ +3 | Length | 2 bytes | 缓冲区长度 |
| +4 | Control | 1 byte | 工作模式、方向、使能等 |
| +5 | Status | 1 byte | 当前状态 |
| +6 | Activate | 1 byte | `0x01` = 激活该 SM |
| +7 | PDI Control | 1 byte | PDI 侧控制 |

### 9.4 SM Control 字节解析

Control 字节（`0x26` / `0x22`）结构：

| 位 | 名称 | 说明 |
|----|------|------|
| bit 0~1 | Operation Mode | `00`=Buffered, `01`=Reserved, `10`=Mailbox, `11`=Unbuffered |
| bit 2 | Direction | `0`=ECAT 读（Input），`1`=ECAT 写（Output） |
| bit 3 | Interrupt ECAT | ECAT 事件中断使能 |
| bit 4 | Watchdog Trigger | Watchdog 触发 |
| bit 5 | Reserved | 保留 |
| bit 6 | PDI Operation Mode | PDI 操作模式 |
| bit 7 | Start Addr > 64KB | 物理地址是否超过 64KB |

本项目邮箱 SM Control：

| SM | Control | 二进制 | 含义 |
|----|---------|--------|------|
| SM0 Out | `0x26` | `0010 0110` | Mailbox 模式，ECAT 写（主站→从站） |
| SM1 In | `0x22` | `0010 0010` | Mailbox 模式，ECAT 读（从站→主站） |

### 9.5 AL Control / AL Status 寄存器

| 地址 | 名称 | 访问 | 说明 |
|------|------|------|------|
| `0x0120` | AL Control | 主站 RW / 从站 R | 主站请求状态切换 |
| `0x0130` | AL Status | 主站 R / 从站 RW | 从站反馈当前状态 |
| `0x0134` | AL Status Code | 主站 R / 从站 RW | 状态转换错误码 |

AL 状态值：

| 值 | 状态 |
|----|------|
| `0x01` | INIT |
| `0x02` | PRE-OP |
| `0x04` | SAFE-OP |
| `0x08` | OPERATIONAL |

本项目中的使用：

- Frame 267 写入 SM 配置时，从站处于 PRE-OP（`0x02`）
- 后续 Frame 283/284 请求 SAFE-OP（`0x04`）
- 后续 Frame 307/308 请求 OP（`0x08`）

### 9.6 本项目涉及的 ESC 寄存器访问路径

```text
EEPROM 读取 SM 地址
    ├── 0x0508 ← 0x1000 (SM0 地址)
    └── 0x0508 ← 0x1080 (SM1 地址)
            │
            ▼
SM 寄存器配置
    ├── 0x0800 ~ 0x0807 ← SM0 配置
    └── 0x0808 ~ 0x080F ← SM1 配置
            │
            ▼
AL Control 状态切换
    ├── 0x0120 ← 0x0002 (PRE-OP)
    ├── 0x0120 ← 0x0004 (SAFE-OP)
    └── 0x0120 ← 0x0008 (OP)
```
