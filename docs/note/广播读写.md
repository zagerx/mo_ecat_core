# 广播读（BRD）与广播写（BWR）分析

## 选取帧

从 `fram.txt` 中选取两对典型帧：

| 命令 | 请求帧 | 响应帧 | 功能 |
|------|--------|--------|------|
| BWR | 3 | 4 | 广播写 AL Control 寄存器 |
| BRD | 7 | 8 | 广播读 ESC Type / Revision |

---

## 一、广播写 BWR —— 以 Frame 3/4 为例

### Frame 3：BWR 请求

```text
No. 3 0.000139146  01:01:01:01:01:01 → Broadcast  ECAT 30
'BWR': Len: 2, Adp 0x0, Ado 0x120, Wc 0
```

展开：

```text
EtherCAT datagram: Cmd: 'BWR' (8), Len: 2, Adp 0x0, Ado 0x120, Cnt 0
    Index: 0x02
    Slave Addr: 0x0000
    Offset Addr: 0x0120
    Length: 2 (0x2)
    AL Ctrl (0x120): 0x0011, Al Ctrl: INIT, Error Ack
    Working Cnt: 0
```

### Frame 4：BWR 响应

```text
No. 4 0.000446267  03:01:01:01:01:01 → Broadcast  ECAT 60
'BWR': Len: 2, Adp 0x1, Ado 0x120, Wc 1
```

展开：

```text
EtherCAT datagram: Cmd: 'BWR' (8), Len: 2, Adp 0x1, Ado 0x120, Cnt 1
    Index: 0x02
    Slave Addr: 0x0001
    Offset Addr: 0x0120
    Length: 2 (0x2)
    AL Ctrl (0x120): 0x0011
    Working Cnt: 1
```

### BWR 机制分析

1. **Cmd = 8 (BWR = Broadcast Write)**
   - 广播写命令，帧会经过链上所有从站

2. **Adp = 0x0000 → 0x0001**
   - 请求帧 `Adp = 0`：表示从链首开始计数
   - 响应帧 `Adp = 1`：说明有 1 个从站处理并响应了该广播命令
   - 每个从站处理完 BWR 后，会把 Adp 加 1

3. **Ado = 0x0120**
   - 寄存器偏移地址，这里是 **AL Control 寄存器**

4. **Data = 0x0011**
   - `0x0001` = INIT 状态请求
   - `0x0010` = Error Ack（错误确认位）

5. **WKC = 0 → 1**
   - 请求帧 WKC = 0
   - 每有一个从站成功执行，WKC +1
   - 本网络只有 1 个从站，所以 WKC = 1

## 二、广播读 BRD —— 以 Frame 7/8 为例

### Frame 7：BRD 请求

```text
No. 7 0.000750601  01:01:01:01:01:01 → Broadcast  ECAT 30
'BRD': Len: 2, Adp 0x0, Ado 0x0, Wc 0
```

展开：

```text
EtherCAT datagram: Cmd: 'BRD' (7), Len: 2, Adp 0x0, Ado 0x0, Cnt 0
    Index: 0x04
    Slave Addr: 0x0000
    Offset Addr: 0x0000
    Length: 2 (0x2)
    Working Cnt: 0
```

### Frame 8：BRD 响应

```text
No. 8 0.001082249  03:01:01:01:01:01 → Broadcast  ECAT 60
'BRD': Len: 2, Adp 0x1, Ado 0x0, Wc 1
```

展开：

```text
EtherCAT datagram: Cmd: 'BRD' (7), Len: 2, Adp 0x1, Ado 0x0, Cnt 1
    Index: 0x04
    Slave Addr: 0x0001
    Offset Addr: 0x0000
    Length: 2 (0x2)
    ESC Revision (0x0): 0xa0
    ESC Type (0x1): 0x02
    Working Cnt: 1
```

### BRD 机制分析

1. **Cmd = 7 (BRD = Broadcast Read)**
   - 广播读命令，所有从站读取同一寄存器地址

2. **Adp = 0x0000 → 0x0001**
   - 与 BWR 相同，表示从链首开始，响应中反映从站数量

3. **Ado = 0x0000**
   - ESC 寄存器起始地址
   - 读取的是 **ESC Type (0x0001)** 和 **ESC Revision (0x0000)**

4. **Data = 0x02a0**
   - 小端解析：
     - 字节 0：`0xa0` → ESC Revision
     - 字节 1：`0x02` → ESC Type
   - 说明该 ESC 类型为 0x02，版本为 0xa0

5. **WKC = 0 → 1**
   - 1 个从站响应，WKC = 1

6. **BRD 的特殊数据合并规则**
   - 多个从站时，每个从站把自己的数据按位与（AND）写回数据区
   - 单站时直接返回该站数据

---

## 三、BWR 与 BRD 对比

| 特性 | BWR（广播写） | BRD（广播读） |
|------|--------------|--------------|
| Cmd | 0x08 | 0x07 |
| 方向 | 主站 → 所有从站 | 主站 ← 所有从站 |
| 从站行为 | 所有从站把数据写入同一寄存器 | 所有从站从同一寄存器读取 |
| Adp 变化 | 每经过一个从站 +1 | 每经过一个从站 +1 |
| WKC 规则 | 每成功写入一个从站 +1 | 每成功读取一个从站 +1 |
| 数据合并 | 无合并，所有从站写相同值 | 多从站时按位与合并 |
| 典型用途 | 初始化寄存器、状态切换 | 扫描 ESC 类型、读取通用状态 |

---

## 四、Adp 字段说明

BWR/BRD 使用 **Auto-increment Address（Adp）**：

- 主站发送时 `Adp = 0`
- 每个从站处理完广播命令后，把 `Adp` 加 1
- 主站通过返回的 `Adp` 值知道网络中有多少个从站

```text
Adp = 1 → 1 个从站
Adp = 3 → 3 个从站
```

> 注意：Adp 在广播命令中表示从站计数，不是站点地址。
