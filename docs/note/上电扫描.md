## 二、ESC 关键寄存器

上电扫描涉及以下 ESC 寄存器（基地址 `0x403A_0000`，来自 RA8T2 手册）：

| 寄存器 | 偏移地址 | 大小 | 读写 | 说明 |
|--------|---------|------|------|------|
| **Type / Revision** | `0x0000` | WORD | R | 从站 ESC 类型和版本号 |
| **Configured Station Address** | `0x0010` | WORD | R/W | **配置站地址**，上电时由 Master 写入 |
| **Configured Station Alias** | `0x0012` | WORD | R/W | 站别名（可选） |
| **DL Status** | `0x0110` | WORD | R | 数据链路状态（拓扑、端口状态） |
| **AL Control** | `0x0120` | WORD | R/W | 应用层控制（请求状态转换） |
| **AL Status** | `0x0130` | WORD | R | 应用层状态（当前状态） |
| **PDI Control** | `0x0140` | WORD | R | PDI 接口类型（SPI/Parallel/...） |
| **SM0 Status** | `0x0805` | BYTE | R | SyncManager 0 状态 |
| **SM1 Status** | `0x080D` | BYTE | R | SyncManager 1 状态 |

> 文档依据：RA8T2 User's Manual 第 36 章，ETG.1000.4 第 6.2 节。

## 三、上电扫描完整流程

上电扫描的核心函数是 SOEM 的 `ecx_config_init()`，它依次完成以下阶段：

```
检测从站数量 → 复位 ESC → 分配 ConfigAdr → 读取详细信息
```

### 3.1 检测总线上有几个从站

**函数**：`ecx_detect_slaves()`（`ec_config.c`）

```c
int ecx_detect_slaves(ecx_contextt *context)
{
    uint16 w;
    int wkc;
    
    // 1. 广播写：让所有从站进入 Init 状态（AL Control = 0x0001）
    w = htoes(EC_STATE_INIT | EC_STATE_ACK);
    ecx_BWR(&context->port, 0x0000, ECT_REG_ALCTL, sizeof(w), &w, EC_TIMEOUTRET3);
    
    // 2. 广播读：读取所有从站的 Type/Revision 寄存器
    //    每个从站响应时 WKC + 1
    wkc = ecx_BRD(&context->port, 0x0000, ECT_REG_TYPE, sizeof(w), &w, EC_TIMEOUTSAFE);
    
    // 3. WKC = 响应的从站数量
    if (wkc > 0)
        context->slavecount = wkc;
    
    return wkc;
}
```

**抓包中看到的帧**：

| 顺序 | CMD | ADP | ADO | 说明 |
|------|-----|-----|-----|------|
| 1 | **BWR** | `0x0000` | `0x0120` | 广播：所有从站复位到 Init |
| 2 | **BRD** | `0x0000` | `0x0000` | 广播读 Type/Revision → WKC = 从站数 |

> **BRD 的 WKC 规则**：每个响应的从站使 WKC + 1。所以返回的 WKC 直接就是从站总数。

---

### 3.2 复位 ESC 内部状态

**函数**：`ecx_set_slaves_to_default()`（`ec_config.c`）

```c
static void ecx_set_slaves_to_default(ecx_contextt *context)
{
    // 广播写：清除各种 ESC 状态
    ecx_BWR(&context->port, 0x0000, ECT_REG_DLPORT, sizeof(b), &b, EC_TIMEOUTRET3);
    ecx_BWR(&context->port, 0x0000, ECT_REG_IRQMASK, sizeof(w), &w, EC_TIMEOUTRET3);
    ecx_BWR(&context->port, 0x0000, ECT_REG_RXERR, 8, &zbuf, EC_TIMEOUTRET3);
    ecx_BWR(&context->port, 0x0000, ECT_REG_FMMU0, 16 * 3, &zbuf, EC_TIMEOUTRET3);  // 清零 FMMU
    ecx_BWR(&context->port, 0x0000, ECT_REG_SM0, 8 * 4, &zbuf, EC_TIMEOUTRET3);     // 清零 SM
    // ...
}
```

**作用**：
- 清零 FMMU 映射（`0x0600` 起）
- 清零 SyncManager 配置（`0x0800` 起）
- 复位错误计数器、IRQ 掩码
- 确保所有从站处于干净的初始状态

**抓包中看到的帧**：

| 顺序 | CMD | ADP | ADO | 说明 |
|------|-----|-----|-----|------|
| 3~N | **BWR** | `0x0000` | `0x0100~` | 广播清零 FMMU、SM、错误计数器等 |

---

### 3.3 给每个从站分配固定地址

**函数**：`ecx_config_init()` 中的循环（`ec_config.c`）

```c
for (slave = 1; slave <= context->slavecount; slave++)
{
    ADPh = (uint16)(1 - slave);  // 计算自增式 ADP
    
    // 1. 读取 PDI 控制寄存器，判断从站接口类型（SPI/Parallel/...）
    val16 = ecx_APRDw(&context->port, ADPh, ECT_REG_PDICTL, EC_TIMEOUTRET3);
    context->slavelist[slave].Itype = etohs(val16);
    
    // 2. 给从站写入固定地址（Configured Station Address）
    //    地址值 = slave + EC_NODEOFFSET（默认偏移 0x1000）
    ecx_APWRw(&context->port, ADPh, ECT_REG_STADR, 
              htoes(slave + EC_NODEOFFSET), EC_TIMEOUTRET3);
    
    // 3. 记录配置地址供后续使用
    context->slavelist[slave].configadr = slave + EC_NODEOFFSET;
}
```

**抓包中看到的帧**（以 2 个从站为例）：

| 顺序 | CMD | ADP | ADO | DATA | 说明 |
|------|-----|-----|-----|------|------|
| N+1 | **APRD** | `0x0000` | `0x0140` | — | 读第 1 个从站的 PDI 类型 |
| N+2 | **APWR** | `0x0000` | `0x0010` | `0x1001` | 给第 1 个从站分配地址 0x1001 |
| N+3 | **APRD** | `0xFFFF` | `0x0140` | — | 读第 2 个从站的 PDI 类型 |
| N+4 | **APWR** | `0xFFFF` | `0x0010` | `0x1002` | 给第 2 个从站分配地址 0x1002 |

> **Configured Station Address 寄存器（0x0010）**：
> - 位置：ESC 基地址 + `0x0010`
> - 大小：WORD (16 bit)
> - 访问：R/W
> - 文档描述：*该参数应包含被配置的从站控制器的站地址。这个站地址在启动时被主站设立。*
> - 依据：ETG.1000.4 第 6.2 节，RA8T2 Manual 第 36.5.1 节

---

### 3.4 读取从站详细信息

分配固定地址后，SOEM 改用 **FPRD/FPWR**（配置式寻址）访问每个从站：

```c
for (slave = 1; slave <= context->slavecount; slave++)
{
    configadr = context->slavelist[slave].configadr;  // 如 0x1001
    
    // 读取 DL Status（拓扑信息、端口状态）
    topology = ecx_FPRDw(&context->port, configadr, ECT_REG_DLSTAT, EC_TIMEOUTRET3);
    
    // 读取 EEPROM 配置（PDO 映射、设备信息）
    // ...
}
```

**抓包中看到的帧**：

| 顺序 | CMD | ADP | ADO | 说明 |
|------|-----|-----|-----|------|
| M+1 | **FPRD** | `0x1001` | `0x0110` | 读 0x1001 从站的 DL Status |
| M+2 | **FPRD** | `0x1001` | `0x0010` | 读 0x1001 从站的配置地址（验证写入成功） |
| M+3 | **FPWR** | `0x1001` | `0x0120` | 写 AL Control，请求进入 Pre-Op |
| ... | ... | ... | ... | 读取 EEPROM、配置 FMMU/SM |

### 4.3 逐帧实验记录

#### Frame 1：BWR DL Status Alias (`0x0103`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | BWR (0x08) | BWR (0x08) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0103` | `0x0103` |
| **Data** | `0x00` | `0x00` |
| **WKC** | `0` | `1` |
| **帧长** | 29 bytes | 60 bytes (Padding) |

**目的**：让所有从站忽略 Configured Station Alias，强制使用物理位置响应。  
**现象**：从站 ESC 硬件直接处理，MCU 固件无感知。回环帧 ADP +1，WKC +1。

---

#### Frame 2：BWR AL Control (`0x0120`) —— 第一次 INIT

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | BWR (0x08) | BWR (0x08) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0120` | `0x0120` |
| **Data** | `0x0011` (INIT + ACK) | `0x0011` |
| **WKC** | `0` | `1` |
| **帧长** | 30 bytes | 60 bytes (Padding) |

**目的**：将所有从站重置到 INIT 状态，并清除错误标志。  
**现象**：与 Frame 1 不同，这帧**会触发 AL Event**（Bit 0 置位）。如果 MCU 固件正在运行 `ECAT_Main()` 轮询，会检测到该事件并调用 `AL_ControlInd(0x11, 0)`。

---

#### Frame 3：BWR AL Control (`0x0120`) —— 重复 INIT（兼容 netX100）

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Data** | `0x0011` (INIT + ACK) | `0x0011` |
| **WKC** | `0` | `1` |

**目的**：兼容旧款 netX100 ESC，确保状态可靠重置。  
**现象**：与 Frame 2 完全相同，从站再次响应 WKC=1。

---

#### Frame 4：BRD ESC Type (`0x0000`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | BRD (0x07) | BRD (0x07) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0000` | `0x0000` |
| **Data** | 空（占位） | `0x00A0` (Rev), `0x0002` (Type) |
| **WKC** | `0` | `1` |
| **帧长** | 30 bytes | 60 bytes (Padding) |

**目的**：检测总线上从站数量。  
**关键发现**：
- **WKC = 1** → 链路上有 1 个从站
- **ESC Type = 0x02** → Renesas R-IN32 系列
- **ESC Revision = 0xA0** → 具体版本

> BRD 与 BWR 的区别：BRD 是**读命令**，从站会将数据**叠加**写入数据区；BWR 是写命令，从站只修改 WKC。

---

#### Frame 5：APRD PDI Control (`0x0140`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | APRD (0x01) | APRD (0x01) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0140` | `0x0140` |
| **Data** | 空（占位） | `0x0080` (PDI), `0x000E` (Ctrl) |
| **WKC** | `0` | `1` |
| **帧长** | 30 bytes | 60 bytes (Padding) |

**目的**：验证 Auto-increment 精确寻址，读取从站的 PDI 接口类型。  
**关键发现**：

```
PDI Ctrl (0x140): 0x80, PDI: On chip bus
PDI Ctrl (0x141): 0x0e, Enhanced link detection, Enable DC sync out, Enable DC latch in
```

- **PDI = On-chip bus**：R-IN32 内部总线，非外部 SPI/并行接口
- **Enhanced Link Detection**：已启用
- **DC Sync Out / Latch In**：已启用

> APRD 与 BRD 的区别：APRD 只有 **ADP 匹配的从站**响应；BRD 是所有从站都响应。单级从站时 WKC 都是 1，多级时差异明显。

---

#### Frame 6：APWR Station Address (`0x0010`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | APWR (0x02) | APWR (0x02) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0010` | `0x0010` |
| **Data** | `0x1001` | `0x1001` |
| **WKC** | `0` | `1` |

**目的**：给从站分配 **Configured Station Address = 0x1001**。  
**关键意义**：
- 分配后，后续帧可用 **FPRD/FPWR**（固定地址命令）直接通过 `0x1001` 寻址该从站
- 不再依赖物理位置（Auto-increment），拓扑变化不影响通信

> APWR 是写命令，从站**不修改数据区**，只递增 WKC。这与 APRD（从站填充数据区）不同。

---

#### Frame 7：APWR DL Control (`0x0100`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | APWR (0x02) | APWR (0x02) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0100` | `0x0100` |
| **Data** | `0x0001` | `0x0001` |
| **WKC** | `0` | `1` |

**目的**：设置第一个从站的 **DL Control**，过滤非 EtherCAT 帧。  
**Wireshark 解析**：

```
ESC Ctrl (0x100): 0x01, Kill non EtherCAT frames
ESC Ctrl (0x101): 0x00, Port 0: Auto loop, Port 1: Auto loop, Port 2: Auto loop, Port 3: Auto loop
```

- **Bit 0 = 1**：从站 1 丢弃非 EtherCAT 帧（如 ARP、DHCP 广播），保证总线实时性
- **Port 0~3 = Auto loop**：端口处于自动环回模式，这是 ESC 的默认状态

---

#### Frame 8：APRD Station Address 验证 (`0x0010`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | APRD (0x01) | APRD (0x01) |
| **ADP** | `0x0000` | `0x0001` |
| **ADO** | `0x0010` | `0x0010` |
| **Data** | `0x0000`（主站清零） | `0x1001`（从站填充） |
| **WKC** | `0` | `1` |

**目的**：验证 Frame 6 写入的 **Configured Station Address = 0x1001** 是否成功。  
**关键意义**：
- 若回环帧 Data = `0x1001`，证明地址分配成功
- `0x1001` 将作为后续 **FPRD/FPWR** 命令的 ConfigAdr
- 这是从 Auto-increment 到 Configured Address 的关键转折点

> APRD 读时，主站发送的数据区被忽略；从站将寄存器值填入数据区。这与 APWR（从站不修改数据区）形成鲜明对比。

---

#### Frame 9：FPRD Alias (`0x0012`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPRD (0x04) | FPRD (0x04) |
| **ADP** | `0x1001` | `0x1001` ⬅️ **不变！** |
| **ADO** | `0x0012` | `0x0012` |
| **Data** | `0x0000`（主站清零） | `0x0000`（从站 Alias） |
| **WKC** | `0` | `1` |

**目的**：用 **Configured Address** 读取从站的 Alias。  
**关键发现**：

```
Phys Addr 2nd (0x12): 0x0000
```

- **Alias = 0x0000**：完全正常。Alias 是可选字段，Renesas `sampleappl` 未在 EEPROM 中配置
- **ADP 不变**：`0x1001` → `0x1001`，证明 Configured Address 模式下 ESC 不修改 ADP
- 这是本实验中第一个 **非自增、非广播** 的命令

> Alias 仅在多从站同型号场景下用于区分，未配置时默认 0。SOEM 将其存入 `slavelist[].aliasadr`，后续 `ecx_config_map` 时用于从站重新匹配。

---

#### Frame 10：FPRD EEPROM Status (`0x0502`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPRD (0x04) | FPRD (0x04) |
| **ADP** | `0x1001` | `0x1001` |
| **ADO** | `0x0502` | `0x0502` |
| **Data** | `0x0000`（主站清零） | `0x8010` / `0x0000` |
| **WKC** | `0` | `1` |

**目的**：读取 EEPROM 控制/状态寄存器，确认 EEPROM 加载状态。  
**关键发现**：

```
EEPROM Control/Status (0x502): 0x8010
    0... .... .... .... = Write access: disabled
    .0.. .... .... .... = Faulty Acknowledge bit: No acknowledge error
    ..0. .... .... .... = Error Acknowledge bit: No error
    ...1 .... .... .... = Busy: Busy
    .... 0... .... .... = EEPROM emulation: disabled
    .... .000 0001 0000 = Command: 0x010 (RELOAD)
```

- **Busy = 1**：ESC 正在加载 EEPROM
- **Command = RELOAD**：表示 EEPROM 加载命令正在执行
- 后续 SOEM 会等待 Busy 清除后再读取 SII 数据

> 在 `ecx_config_init()` 中，SOEM 通过读取 EEPROM Status 判断是否可以安全读取 SII 内容。

---

### 4.4 关键现象总结

#### 现象 1：发送帧 29/30 bytes vs 回环帧 60 bytes

| | 发送帧 | 回环帧 |
|---|---|---|
| **长度** | 29/30 bytes | 60 bytes |
| **原因** | RAW SOCKET 直接发送，无填充 | 从站 PHY (KSZ8081) 执行 802.3 最小帧长填充 |
| **Padding** | 无 | 30 bytes (全 0) |

> 以太网最小帧长 = 64 bytes（含 FCS）/ 60 bytes（不含 FCS）。RAW SOCKET 绕过了内核的帧长检查，但标准 PHY 会强制执行。

#### 现象 2：ADP 变化规则

| 命令类型 | 发送帧 ADP | 回环帧 ADP | 变化原因 |
|---|---|---|---|
| BWR/BRD | `0x0000` | `0x0001` | 广播命令，每经过一从站 +1 |
| APRD/APWR | `0x0000` | `0x0001` | 自增命令，命中从站后 +1 |
| **FPRD/FPWR** | **`0x1001`** | **`0x1001`** | **配置地址命令，ESC 不修改 ADP** |

> **FPRD 的 ADP 不变** 是 Configured Address 寻址的核心特征：ESC 直接将 ADP 与自身 Station Address 寄存器（`0x0010`）比对，匹配则处理，不匹配则透传。

所有 Auto-increment 命令（APRD/APWR）和广播命令（BWR/BRD）都遵循同一规则：
> **每个从站经过时，ADP 自动 +1。**

单级从站时，回环帧 ADP 总是 `0x0001`；N 级从站时，回环帧 ADP = `0x000N`。

#### 现象 3：Source MAC 从 `01:01:01:01:01:01` → `03:01:01:01:01:01`

回环帧的第一个字节从 `0x01` 变成 `0x03`（Bit 1 置位），这是从站 ESC 硬件在回环时自动修改的，**不影响 EtherCAT 通信**（SOEM 不校验 Source MAC）。

#### 现象 4：状态的返回不在同一帧中

BWR/APWR 写入 AL Control (`0x0120`) 后，AL Status 的更新发生在从站内部（`SetALStatus()` 写 ESC `0x130` 寄存器），但主站**不会自动读到**。主站需要额外发送 **APRD/BRD** 读取 `0x0130` 才能获取状态。

---

### 4.5 命令类型对比

| 命令 | 类型 | 寻址方式 | 数据区变化 | WKC 规则 | 本实验中用途 |
|---|---|---|---|---|---|
| **BWR** | 广播写 | 所有从站处理 | 原样返回 | 每站 +1 | 初始化、重置状态 |
| **BRD** | 广播读 | 所有从站处理 | 从站叠加写入 | 每站 +1 | 检测从站数量 |
| **APRD** | 自增读 | ADP=0 的从站处理 | 从站填充数据 | 命中站 +1 | 读 PDI Control |
| **APWR** | 自增写 | ADP=0 的从站处理 | 原样返回 | 命中站 +1 | 分配 Station Address |
| **FPRD** | 固定地址读 | ADP=ConfigAdr 的从站处理 | 从站填充数据 | 命中站 +1 | 读 Alias、EEP Status |
| **FPWR** | 固定地址写 | ADP=ConfigAdr 的从站处理 | 原样返回 | 命中站 +1 | 写 AL Control、Mailbox |

## 五、SDO/Mailbox 通信实验

### 5.1 实验目的

在上一章完成上电扫描的基础上，进一步验证主站如何通过 **Mailbox (SM0/SM1)** 与从站进行 **CoE/SDO** 协议通信。本实验读取对象字典 `0x1018:1`（Vendor ID），并与 EEPROM 读取值对比。

### 5.2 核心发现：Mailbox 必须写满整个 Buffer

#### 5.2.1 修改前后的对比

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 写入长度 | 16 bytes | **128 bytes** |
| FPWR WKC | 1 | 1 |
| FPRD WKC | **0** | **1** ✅ |
| 响应 | 无 | `0x43` (SDO Upload Response, 4 bytes) |
| Vendor ID | - | `0x00000766` ✅ |

#### 5.2.2 为什么必须写 128 字节？

R-IN32 ESC 的 SyncManager 在 **Mailbox 模式** 下使用硬件 **Buffer Full/Empty** 握手：

```
主站写 SM0 ──→ ESC 监控写入 ──→ 检测到完整 128B 写入？
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
               是（写满 128B）      否（只写 16B）
                    │                   │
                    ▼                   ▼
            设置 SM0 Status            不设置
            Buffer Full = 1            Buffer Full = 0
                    │                   │
                    ▼                   ▼
            生成 MAILBOX_WRITE        无事件
            EVENT (AL Event bit2)
                    │                   │
                    ▼                   ▼
            从站检测到事件            从站无感知
            读取并处理请求            不处理请求
```

**只写 16 字节时**，ESC 认为"主站还没写完"，于是：
- ❌ 不设置 SM0 Buffer Full
- ❌ 不生成 MAILBOX_WRITE_EVENT
- ❌ 从站 `ECAT_Main()` 永远轮询不到事件
- ❌ `MBX_CheckAndCopyMailbox()` 不被调用
- ❌ SM1 永远为空 → 主站 FPRD WKC=0

> **SOEM 的 `ecx_mbxsend()` 始终写入 `mbx_l`（128B），并将未使用部分清零。SSC 从 ESC 读取时只读 `Length + Header` 字节，后面的零不影响解析。**

#### 5.2.3 128 字节内部结构

```
地址      数据
0x1000    0A 00 00 00 00 13 00 20 40 18 10 01 66 07 00 00   ← 16 字节有效数据
0x1010    00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00   ← 全零
0x1020    00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00   ← 全零
...       ...
0x1070    00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00   ← 全零
          ↑ 到 0x107F 结束（共 128 字节）
```

**前 16 字节逐字段解析**：

| 偏移 | 值 | 字段 | 含义 |
|------|-----|------|------|
| 0-1 | `0A 00` | `Length` | Mailbox 数据部分长度 = 10 字节 |
| 2-3 | `00 00` | `Address` | 保留 |
| 4 | `00` | `Priority` | 优先级 |
| 5 | `13` | `Type+Cnt` | CoE 协议 (0x03) + Counter=1 |
| 6-7 | `00 20` | `CoE Header` | SDO Request (0x02 << 12) |
| 8 | `40` | `SDO Command` | Upload Request |
| 9-10 | `18 10` | `Index` | 0x1018 (Vendor ID 对象) |
| 11 | `01` | `SubIndex` | 子索引 1 |
| 12-15 | `00 00 00 00` | `Data` | Upload Request 无数据 |

### 5.3 逐帧解析 Frame 11-14

#### Frame 11：FPWR AL Control = PRE-OP (`0x0120`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPWR (0x05) | FPWR (0x05) |
| **ADP** | `0x1001` | `0x1001` |
| **ADO** | `0x0120` | `0x0120` |
| **Data** | `0x0002` (PRE-OP) | `0x0002` |
| **WKC** | `0` | `1` |

**目的**：请求从站切换到 **PRE-OPERATIONAL** 状态。  
**从站行为**：
- ESC 收到 AL Control = 0x0002
- 生成 AL Event，MCU 检测到后调用 `AL_ControlInd()`
- 启动 Mailbox Handler：`MBX_StartMailboxHandler()`
- 配置 SM0/SM1 并使能
- 设置 `AL Status = 0x0002` 表示已进入 PRE-OP

---

#### Frame 12：FPRD AL Status (`0x0130`)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPRD (0x04) | FPRD (0x04) |
| **ADP** | `0x1001` | `0x1001` |
| **ADO** | `0x0130` | `0x0130` |
| **Data** | `0x0000`（主站清零） | `0x0002`（从站状态） |
| **WKC** | `0` | `1` |

**目的**：验证从站是否已成功进入 PRE-OP。  
**关键发现**：
- 回环帧 Data = `0x0002` → 从站已在 PRE-OP 状态
- 若最高位为 1（如 `0x0012`），则表示有错误

---

#### Frame 13：FPWR Mailbox Out (`0x1000`, 128B)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPWR (0x05) | FPWR (0x05) |
| **ADP** | `0x1001` | `0x1001` |
| **ADO** | `0x1000` | `0x1000` |
| **Len** | 128 | 128 |
| **Data** | 128B SDO Upload Request | 原样返回 |
| **WKC** | `0` | `1` |
| **Wireshark** | `Mbx(CoE SDO Req : 'Initiate Upload' (2) Idx=0x1018 Sub=1)` | 同上 |

**目的**：将 SDO Upload Request 写入从站 **SM0 (Mailbox Out)**。  
**关键条件**：必须写入 **完整 128 字节**，否则 ESC 不触发 MAILBOX_WRITE_EVENT。

---

#### Frame 14：FPRD Mailbox In (`0x1080`, 128B)

| 字段 | 发送帧 | 回环帧 |
|---|---|---|
| **Cmd** | FPRD (0x04) | FPRD (0x04) |
| **ADP** | `0x1001` | `0x1001` |
| **ADO** | `0x1080` | `0x1080` |
| **Len** | 128 | 128 |
| **Data** | 空（占位） | 128B SDO Upload Response |
| **WKC** | `0` | `1` |
| **Wireshark** | — | `Mbx(CoE SDO Res: Scs 2)` |

**目的**：从 **SM1 (Mailbox In)** 读取从站的 SDO 响应。  
**响应帧前 16 字节**：

```
0A 00 00 00 00 13 00 30 43 18 10 01 66 07 00 00
```

| 偏移 | 字节 | 字段 | 值 |
|------|-----|------|-----|
| 0-1 | `0A 00` | `Length` | 10 |
| 5 | `13` | `MbxType` | CoE + Cnt=1 |
| 6-7 | `00 30` | `CoE Header` | SDO Response (0x03 << 12) |
| 8 | `43` | `Command` | Upload Response, Expedited, 4 bytes |
| 9-10 | `18 10` | `Index` | 0x1018 |
| 11 | `01` | `SubIndex` | 0x01 |
| 12-15 | `66 07 00 00` | `Data` | `0x00000766` (Vendor ID) |

### 5.4 从站处理流程

```

  └─ ECAT_Main()

                                                              └─ MBX_CopyToSendMailbox()

```

## 六、常见问题

### Q1：Alias = 0x0000 是否正常？

**完全正常。**

- Alias 是 ESC 寄存器 `0x0012` 中的可选第二地址
- Renesas `sampleappl` 未在 EEPROM 中配置 Alias
- 单/多数从站场景下 Alias 为 0 不影响任何功能
- 只有在多同型号从站需要区分时才配置非零 Alias

### Q2：为什么发送帧 29/30 bytes，回环帧 60 bytes？

- 发送帧通过 RAW SOCKET 直接发送，绕过了内核的帧长检查
- 回环帧经过标准 PHY (KSZ8081)，PHY 强制执行 802.3 最小帧长 = 60 bytes（不含 FCS）
- 多出的 30 bytes 是全 0 Padding

### Q3：为什么 FPWR 写入 16 字节时从站不响应？

- R-IN32 ESC 的 Mailbox SyncManager 要求**完整 buffer 写入**才触发事件
- SM0 大小配置为 128 字节，必须写满 128 字节
- 只写 16 字节时，ESC 认为写入未完成，不生成 MAILBOX_WRITE_EVENT
- 从站收不到事件，不处理请求，SM1 保持为空

### Q4：主站怎么知道 SM0/SM1 的地址和大小？

- 主站在 `ecx_config_init()` 中读取从站 EEPROM 的 SII
- `SII[0x0018]` = SM0 起始地址 + 大小
- `SII[0x001A]` = SM1 起始地址 + 大小
- 这些值由从站厂商在 EEPROM 中配置

### Q5：对象字典 0x1018 是从站通过 EEPROM 告诉主站的吗？

**不是。**

- EEPROM 只存从站身份和 Mailbox/PDO 配置
- 完整的对象字典通过 **ESI 文件**（离线配置）或 **SDO Info**（在线查询）获取
- 运行时主站通过 SDO 直接访问已知对象索引
