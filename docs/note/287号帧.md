# 287 号帧分析

## 帧头信息

```text
No. 287 0.050293341  01:01:01:01:01:01 → Broadcast  ECAT 36
'LRW': Len: 8, Adp 0x0, Ado 0x0, Wc 0
```

| 字段 | 值 | 说明 |
|------|-----|------|
| Cmd | `LRW` (12) | 逻辑读写命令 |
| Len | 8 | 数据长度 8 bytes |
| Adp | `0x0000` | 广播地址，不寻址单个从站 |
| Ado / Addr | `0x00000000` | **逻辑地址**，不是 ESC 寄存器地址 |
| Wc | 0 | 请求帧，响应中从站会递增 |

## LRW 命令含义

**LRW = Logical Read Write**，一条命令同时完成：

1. **写操作**：把帧中的数据写入逻辑地址对应的输出区域
2. **读操作**：从逻辑地址对应的输入区域读取数据，覆盖帧中对应位置

## 逻辑地址与 FMMU 映射

287 帧访问的逻辑地址是 `0x00000000`，长度 8 bytes：

```text
逻辑地址 0x00000000 ~ 0x00000007
```

根据 FMMU 配置，这段地址被映射为：

| 逻辑地址范围 | FMMU | 物理地址 | 方向 | 数据 |
|--------------|------|----------|------|------|
| `0x0000 ~ 0x0003` | FMMU0 | `0x1100 ~ 0x1103` (SM2) | Output | LED 输出 |
| `0x0004 ~ 0x0007` | FMMU1 | `0x1400 ~ 0x1403` (SM3) | Input | DipSw 输入 |

## 8 bytes 数据含义

287 帧请求中的数据段（8 bytes）：

```text
前 4 bytes: 输出数据 → 经 FMMU0 映射到 SM2 (0x1100)
后 4 bytes: 任意值（占位）→ 经 FMMU1 映射被从站输入数据覆盖
```

程序中发送前：

```c
pd[0] = led_value;   // LED 输出值
pd[1] = 0;
pd[2] = 0;
pd[3] = 0;
pd[4~7] = 0;         // 输入占位，从站会覆盖
ecx_LRW(&ctx.port, PDO_LOG_ADDR, PDO_SIZE, pd, EC_TIMEOUTRET);
```

## 为什么 Adp=0

LRW 使用**逻辑地址寻址**，不针对单个从站的站点地址。所有从站 ESC 都会检查逻辑地址是否命中自己的 FMMU：

- 命中 → 读写对应物理地址，并递增 WKC
- 未命中 → 直接转发，不递增 WKC

所以 `Adp=0x0000` 表示广播/链式转发，ESC 根据逻辑地址自行判断是否响应。

## 响应帧 288

```text
No. 288 0.050567075  03:01:01:01:01:01 → Broadcast  ECAT 60
'LRW': Len: 8, Adp 0x0, Ado 0x0, Wc 3
```

| 字段 | 值 | 说明 |
|------|-----|------|
| Wc | 3 | 输出写命中 +1，输入读命中 +2，共 +3 |

288 帧返回时：

- 前 4 bytes：保持输出值（从站不修改输出区）
- 后 4 bytes：被替换为从站 `0x1400` 处的输入数据（DipSw）

## LRW 下发频率

### 决定因素

| 因素 | 说明 |
|------|------|
| 控制任务周期 | PLC 任务周期、运动控制周期等 |
| SM watchdog | LRW 周期必须小于 watchdog 超时 |
| 总线带宽 | 周期越短，总线负载越高 |
| 实时性需求 | 闭环控制要求周期稳定、抖动小 |

常见周期：

| 周期 | 典型应用 |
|------|----------|
| 125 µs | 高端运动控制 |
| 1 ms | 伺服、机器人 |
| 4 ms / 8 ms | 普通 PLC I/O |
| 10 ms / 100 ms | 监控、指示灯 |

### 如何设定

在 SOEM 中通过主循环的 `osal_usleep()` 控制：

```c
#define LOOP_PERIOD_US  100000   /* 100 ms */

while (g_run)
{
    ecx_send_processdata(&ctx);
    ecx_receive_processdata(&ctx, EC_TIMEOUTRET);
    osal_usleep(LOOP_PERIOD_US);
}
```

本项目 `pdo_manual.c` 当前设置为 **100 ms**。

### 注意事项

- 周期必须稳定，抖动过大可能导致 WKC 异常或从站掉出 OP
- 从 SAFE-OP → OP 前需要快速连续发送几帧 LRW 喂 watchdog
- `ecx_receive_processdata()` 的 timeout 应小于 LRW 周期
