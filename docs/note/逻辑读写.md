# 逻辑读（LRD）、逻辑写（LWR）、逻辑读写（LRW）分析

## 当前抓包情况

在 `fram.txt` 中：

- **LRW 帧**：大量存在（从 Frame 287 开始）
- **LRD 帧**：未找到
- **LWR 帧**：未找到

原因是本项目 `pdo_manual.c` 使用 `ecx_LRW()` 同时完成 PDO 输出写入和输入读取，因此只需要 LRW 一种命令即可。

---

## 一、LRW 实际帧分析 —— 以 Frame 287/288 为例

### Frame 287：LRW 请求

```text
No. 287 0.050293341  01:01:01:01:01:01 → Broadcast  ECAT 36
'LRW': Len: 8, Adp 0x0, Ado 0x0, Wc 0
```

展开：

```text
EtherCAT datagram: Cmd: 'LRW' (12), Len: 8, Addr 0x0, Cnt 0
    Index: 0x00
    Log Addr: 0x00000000
    Length: 8 (0x8)
    Data: 0000000000000000
    Working Cnt: 0
```

> LRW 使用 **Log Addr（逻辑地址）**，不是 Slave Addr + Offset Addr。

### Frame 288：LRW 响应

```text
No. 288 0.050567075  03:01:01:01:01:01 → Broadcast  ECAT 60
'LRW': Len: 8, Adp 0x0, Ado 0x0, Wc 3
```

展开：

```text
EtherCAT datagram: Cmd: 'LRW' (12), Len: 8, Addr 0x0, Cnt 3
    Index: 0x00
    Log Addr: 0x00000000    ← 逻辑地址不变
    Length: 8 (0x8)
    Data: 00000000f8fed49e
    Working Cnt: 3
```

### LRW 机制分析

1. **Cmd = 12 (LRW = Logical Read Write)**
   - 一条命令同时完成逻辑地址空间的读和写

2. **Addr = 0x00000000**
   - 逻辑起始地址，不是 ESC 物理地址
   - ESC 通过 FMMU 表映射到物理地址

3. **Len = 8**
   - 访问 8 bytes 逻辑地址空间
   - 前 4 bytes：输出（FMMU0 → SM2 0x1100）
   - 后 4 bytes：输入（FMMU1 → SM3 0x1400）

4. **WKC = 0 → 3**
   - 输出 FMMU0 命中：+1
   - 输入 FMMU1 命中：+2
   - 期望 WKC = 3

5. **数据流向**

```text
主站发送: [LED输出 4B][00 00 00 00]
              │              │
              ▼              ▼
           FMMU0          FMMU1
              │              │
              ▼              ▼
           SM2 0x1100    SM3 0x1400
           写入 LED      读取 DipSw
              │              │
              ▼              ▼
从站返回: [LED输出 4B][DipSw输入 4B]
```

## 二、LRD / LWR / LRW 命令对比

虽然当前抓包中没有 LRD 和 LWR，但这三种命令在 EtherCAT 中各有用途：

| 命令 | Cmd | 名称 | 操作 | 典型应用 |
|------|-----|------|------|----------|
| LRD | 0x09 | Logical Read | 只读逻辑地址 | 读取多个从站输入 PDO |
| LWR | 0x0A | Logical Write | 只写逻辑地址 | 写入多个从站输出 PDO |
| LRW | 0x0C | Logical Read Write | 同时读写 | 单个从站输出+输入同时刷新 |

### 使用场景

- **LWR 单独使用**：
  - 只有输出数据，没有输入数据时
  - 例如：只控制 LED，不读取传感器

- **LRD 单独使用**：
  - 只有输入数据，没有输出数据时
  - 例如：只读取传感器状态

- **LRW 使用**：
  - 同时有输出和输入时
  - 效率最高，一个帧完成读写
  - 本项目的 PDO 通信就是这种情况

### WKC 规则

| 命令 | 单输出 FMMU | 单输入 FMMU | 输出+输入各 1 个 |
|------|------------|------------|----------------|
| LWR | +1 | 0 | +1 |
| LRD | 0 | +1 | +1 |
| LRW | +1 | +2 | +3 |

> 输入 FMMU 的 WKC 权重是 +2，因为 LRW 对输入区域既有读操作，也有写回操作。

## 三、为什么本项目不用 LRD/LWR

本项目 PDO 映射：

- 输出 PDO：`0x7000` → 4 bytes
- 输入 PDO：`0x6000` → 4 bytes

如果使用 LRD/LWR 需要两帧：

```text
Frame A: LWR Addr=0x00000000 Len=4  → 写 LED 输出
Frame B: LRD Addr=0x00000004 Len=4  → 读 DipSw 输入
```

使用 LRW 只需一帧：

```text
Frame: LRW Addr=0x00000000 Len=8  → 同时写 LED、读 DipSw
```

因此 LRW 更高效，是 PDO 通信的常用方式。

---

## 四、如何生成 LRD/LWR 抓包

如果需要实际观察 LRD/LWR 帧，可以写一个简单的测试程序：

```c
/* 只写输出 */
ecx_LWR(&ctx.port, 0x00000000, 4, out_buf, EC_TIMEOUTRET);

/* 只读输入 */
ecx_LRD(&ctx.port, 0x00000004, 4, in_buf, EC_TIMEOUTRET);
```

> 注意：使用 LRD/LWR 时，需要确保 FMMU 配置中的逻辑地址与访问地址一致。
