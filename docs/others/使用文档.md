# mo_ecat_core 使用文档

## 一、获取代码并初始化子模块

```bash
git submodule update --init --recursive
```

## 二、一键编译

项目提供 `examples/build.sh`，会把核心库和 CLI 统一编译到 `examples/cli/build/` 下：

```bash
./examples/build.sh
```

产物：

- `examples/cli/build/mo_ecat_core/libmo_ecat_core.a`
- `examples/cli/build/ecat_cli`

## 三、手动编译

过渡期 CLI 的 `CMakeLists.txt` 通过 `add_subdirectory` 引入了仓库根目录的核心库，因此只需要配置 `examples/cli/`：

```bash
cd examples/cli
cmake -B build .
cmake --build build -j$(nproc)
```

所有产物都在 `examples/cli/build/` 中。

## 四、运行 CLI

```bash
sudo ./examples/cli/build/ecat_cli enp0s31f6
```

> 把 `enp0s31f6` 替换成实际的网卡名。程序启动后进入 `AdapterReady` 状态，需要在交互命令中依次执行 `scan`、`config`、`prepare`、`start` 才会尝试进入 `OPERATIONAL`。按 `Ctrl+C` 可退出。

## 五、在其他项目中使用核心库

核心库根目录的 `CMakeLists.txt` 仍然可以单独构建，并导出 `mo_ecat_core` 包配置：

```bash
cmake -B build .
cmake --build build -j$(nproc)
```

外部项目通过 `find_package(mo_ecat_core REQUIRED)` 引入，构建时指定 `CMAKE_PREFIX_PATH` 指向核心库的构建目录或安装目录。
