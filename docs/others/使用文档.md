# mo_ecat_core 使用文档

## 一、获取代码并初始化子模块

```bash
git submodule update --init --recursive
```

## 二、一键编译

项目提供 `examples/build.sh`，会同时编译核心库和 CLI，所有构建产物都在 `examples/` 下：

```bash
./examples/build.sh
```

产物：

- `examples/build/libmo_ecat_core.a`
- `examples/build/mo_ecat_coreConfig.cmake`
- `examples/cli/build/ecat_cli`

## 三、手动编译

### 编译核心库

根目录的 `CMakeLists.txt` 只负责构建核心库。为了把构建产物集中到 `examples/` 下：

```bash
cmake -B examples/build .
cmake --build examples/build -j$(nproc)
```

### 编译 CLI 示例

```bash
cd examples/cli
cmake -B build -DCMAKE_PREFIX_PATH=../build
cmake --build build -j$(nproc)
```

## 四、运行 CLI

```bash
sudo ./examples/cli/build/ecat_cli enp0s31f6
```

> 把 `enp0s31f6` 替换成实际的网卡名。程序启动后进入 `AdapterReady` 状态，需要在交互命令中依次执行 `scan`、`config`、`prepare`、`start` 才会尝试进入 `OPERATIONAL`。按 `Ctrl+C` 可退出。

## 五、在其他项目中使用核心库

```cmake
find_package(mo_ecat_core REQUIRED)
add_executable(my_app main.cpp)
target_link_libraries(my_app PRIVATE mo_ecat_core)
```

构建时通过 `CMAKE_PREFIX_PATH` 指向 `mo_ecat_core` 的构建目录或安装目录。

## 六、编写 PDO 测试程序

如果需要测试 PDO 读写，可以新建一个可执行文件并通过 `find_package(mo_ecat_core)` 链接：

```cmake
find_package(mo_ecat_core REQUIRED)
add_executable(pdo_test pdo_test.cpp)
target_link_libraries(pdo_test PRIVATE mo_ecat_core)
```
