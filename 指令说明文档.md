# 主站 CLI 指令说明文档

本文档说明 `MoDriverPC_EtherCAT` 主站程序在交互式命令行中支持的所有指令。

> 适用程序：`ecat_scan`（构建产物）  
> 启动方式：`./ecat_scan <网卡名>`，例如 `./ecat_scan eth0`

---

## 1. 基本概念：主站生命周期状态

主站内部维护一个状态机，**大部分指令只能在特定状态下执行**。状态顺序为：

```
Uninitialized -> AdapterReady -> Scanned -> Maintenance -> Operational
                                      ^                    |
                                      |                    v
                                      +------------------ Error
```

| 状态 | 含义 |
|------|------|
| `AdapterReady` | 网卡/SOEM 初始化完成，尚未扫描从站 |
| `Scanned` | 已扫描到从站，尚未进入 PREOP |
| `Maintenance` | 所有从站已进入 PREOP，可执行 SDO 维护类命令 |
| `Operational` | 所有从站已进入 OP，PDO 周期运行中 |
| `Error` | 发生不可恢复异常，只能 `stop` |

---

## 2. 指令总览

| 指令 | 可用状态 | 作用 |
|------|----------|------|
| `scan` | AdapterReady | 扫描 EtherCAT 总线上的从站 |
| `config` | Scanned | 请求从站进入 PREOP，并创建 SlaveNode 占位 |
| `diagnose` | Maintenance / Operational | 对所有从站执行 SDO 诊断 |
| `param <index> <subindex> <value>` | Maintenance / Operational | 对所有从站写入一个 SDO 参数 |
| `inspect` | Maintenance / Operational | 巡检所有从站的实时状态和 AL status code |
| `pdo [slave_id]` | Maintenance / Operational | 读取 PDO 映射 |
| `start` | Maintenance | 配置 PDO → SAFEOP → DC → OPERATIONAL |
| `stop` | 任意 | 安全关闭主站，回到 Uninitialized |
| `loglevel <level>` | 任意 | 设置日志级别 |
| `exit` / `quit` / Ctrl+D | 任意 | 退出程序 |
| `help` | 任意 | 打印帮助信息 |

---

## 3. 指令详解

### 3.1 `scan`

- **语法**：`scan`
- **可用状态**：`AdapterReady`
- **作用**：扫描 EtherCAT 总线，读取从站 Vendor / Product / Revision 等信息，并校验配置的期望从站数量和身份。
- **成功输出示例**：

```text
[INFO] EcatController -> Scanned, 1 slave(s)
```

- **失败处理**：如果从站数量或身份不匹配，或总线上无从站，主站会关闭 SOEM 并回到 `Uninitialized`。

---

### 3.2 `config`

- **语法**：`config`
- **可用状态**：`Scanned`
- **作用**：请求所有从站进入 `PREOP` 状态，并为每个从站创建一个 `SlaveNode` 占位对象。
- **成功输出示例**：

```text
[INFO] Slave 1 [Joint1] placeholder created (vendor=0x766, product=0x2000)
[INFO] EcatController -> Maintenance, 1 slave node(s) placeholder created
```

- **注意**：当前 `config` 不会真正配置从站的 PDO 映射，只是进入 PREOP 并为后续 SDO 维护做准备。

---

### 3.3 `diagnose`

- **语法**：`diagnose`
- **可用状态**：`Maintenance` / `Operational`
- **作用**：对所有从站执行 SDO 诊断，读取标准对象：
  - `0x1000:00`：Device type
  - `0x1018:01`：Vendor ID
  - `0x1018:02`：Product code
  - `0x1018:03`：Revision number
- **成功输出示例**：

```text
[INFO] SdoDiagnostics passed for slave 1 [Joint1]: device_type=0x..., vendor_id=0x..., product_code=0x..., revision=0x...
```

- **失败处理**：诊断失败不会破坏主站状态，仅在日志中记录警告。

---

### 3.4 `param`

- **语法**：`param <index> <subindex> <value>`
- **可用状态**：`Maintenance` / `Operational`
- **作用**：对所有从站写入一个 SDO 参数，写入后回读校验。
- **参数说明**：
  - `index`：对象索引，支持十进制或十六进制（如 `0x6060`）。
  - `subindex`：子索引，支持十进制或十六进制（如 `0x00`）。
  - `value`：要写入的值，支持十进制或十六进制（如 `0x08`）。
- **示例**：

```text
param 0x6060 0x00 0x08
```

- **成功输出示例**：

```text
[INFO] Writing parameter 0x6060:0 = 0x8
[INFO] SdoParameter verified for slave 1 [Joint1]: read_back=0x8
[INFO] SdoParameter finished successfully for slave 1 [Joint1]
```

- **注意事项**：
  - 当前实现**固定写入 4 字节**（`uint32_t`），请确保目标对象长度与之匹配。
  - 写入失败默认会导致主站进入 `Error` 状态。

---

### 3.5 `inspect`

- **语法**：`inspect`
- **可用状态**：`Maintenance` / `Operational`
- **作用**：实时读取所有从站的当前 EtherCAT 状态和 AL status code。
- **输出示例**：

```text
[INFO] StateInspection for slave 1 [Joint1]: state=0x12 (PREOP), al_status_code=0x0
```

- **状态说明**：
  - `0x01`：INIT
  - `0x02`：PREOP
  - `0x04`：SAFEOP
  - `0x08`：BOOT
  - `0x12`：OP（含错误标志）

---

### 3.6 `pdo`

- **语法**：
  - `pdo`：读取所有从站的 PDO 映射
  - `pdo <slave_id>`：读取指定从站的 PDO 映射
- **可用状态**：`Maintenance` / `Operational`
- **作用**：通过 SDO 读取从站的 PDO 分配和映射：
  - `0x1C12`：RxPDO 分配
  - `0x1C13`：TxPDO 分配
  - `0x1600 ~ 0x17FF`：RxPDO 映射
  - `0x1A00 ~ 0x1BFF`：TxPDO 映射
- **输出示例**：

```text
PDO mapping for slave 1 [Joint1] RxPDO (0x1c12):
  PDO 0x1600:
    [0] 0x6040:0 (16 bits)
PDO mapping for slave 1 [Joint1] TxPDO (0x1c13):
  PDO 0x1a00:
    [0] 0x6041:0 (16 bits)
```

- **说明**：输出中每个条目的格式为 `0x<index>:<subindex> (<bit_length> bits)`。

---

### 3.7 `start`

- **语法**：`start`
- **可用状态**：`Maintenance`
- **作用**：启动 PDO 周期运行。内部按顺序执行：
  1. 配置 SlaveNode（`DoPdoConfigure`）
  2. 配置过程数据并请求 `SAFEOP`
  3. 配置分布式时钟（DC）
  4. 请求 `OPERATIONAL`
- **成功输出示例**：

```text
[INFO] PDO configured
[INFO] SAFEOP reached
[INFO] DC configured
[INFO] EcatController -> Operational
```

- **失败处理**：任意子步骤失败都会使主站进入 `Error` 状态。
- **注意**：进入 `Operational` 后，只允许执行 `stop`；其它维护命令会被拒绝。

---

### 3.8 `stop`

- **语法**：`stop`
- **可用状态**：任意
- **作用**：安全停止主站，根据当前状态逐级回滚：
  - `Operational` → SAFEOP → PREOP → INIT → 关闭
  - `Maintenance` → PREOP → INIT → 关闭
- **效果**：最终回到 `Uninitialized`，释放网卡和 SOEM 资源。

---

### 3.9 `loglevel`

- **语法**：`loglevel debug|info|warn|error`
- **可用状态**：任意
- **作用**：动态设置日志输出级别。
- **示例**：

```text
loglevel debug
```

- **输出示例**：

```text
[INFO] Log level set to debug
```

---

### 3.10 `exit` / `quit` / Ctrl+D

- **语法**：`exit` 或 `quit`，也可以在标准输入上按 `Ctrl+D`。
- **可用状态**：任意
- **作用**：请求主程序退出。程序会先调用 `stop` 清理资源，然后返回。

---

### 3.11 `help`

- **语法**：`help`
- **可用状态**：任意
- **作用**：打印当前可用的命令列表。

---

## 4. 典型操作流程

以下是一个完整的最小操作示例：

```text
# 1. 启动程序后进入 AdapterReady
> scan
[INFO] EcatController -> Scanned, 1 slave(s)

# 2. 进入 Maintenance（PREOP）
> config
[INFO] EcatController -> Maintenance, 1 slave node(s) placeholder created

# 3. 查看 PDO 映射
> pdo
[INFO] PDO mapping for slave 1 [Joint1] RxPDO (0x1c12):
  PDO 0x1600:
    [0] 0x6040:0 (16 bits)
[INFO] PDO mapping for slave 1 [Joint1] TxPDO (0x1c13):
  PDO 0x1a00:
    [0] 0x6041:0 (16 bits)

# 4. 写入参数（例如设置运行模式为 CSP）
> param 0x6060 0x00 0x08

# 5. 启动 PDO 周期
> start
[INFO] EcatController -> Operational

# 6. 停止并退出
> stop
> exit
```

---

## 5. 注意事项

1. **状态严格受限**：在 `Operational` 状态下，除 `stop`、`loglevel`、`exit`、`help` 外，其它命令会被拒绝。
2. **Activity 串行执行**：`diagnose`、`param`、`inspect`、`pdo` 等命令会同步执行一个 Activity；如果前一个 Activity 尚未结束，新命令会提示 `Activity already running`。
3. **`param` 固定 4 字节**：当前实现对所有对象都写入 4 字节，向 1 字节或 2 字节对象写入可能导致错误。
4. **`start` 不可逆**：一旦失败进入 `Error`，只能 `stop` 后重新 `scan` → `config` → `start`。
5. **Ctrl+C 也会安全退出**：程序捕获 `SIGINT` / `SIGTERM`，会完成清理后再退出。
