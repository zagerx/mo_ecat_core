# EcatController 状态机说明文档

## 1. 概述

`EcatController` 是 EtherCAT 主站生命周期控制器，内部维护一个有限状态机（FSM），用于管理主站从网卡绑定到 OP 运行的完整流程，以及异常时的错误状态表达。

所有状态转换都通过统一入口 `TransitionTo(ControllerState target)` 完成，集中处理前置条件检查、转换执行、失败回滚和日志记录。

当前阶段（PREOP + SDO）主站生命周期走到 `kMaintenance` 即稳定，后续通过命令 `start` 或上层调用 `StartOperation()` 进入 PDO 阶段。

---

## 2. 状态定义

```cpp
enum class ControllerState {
    kUninitialized,   // 初始状态：未绑定网卡、未扫描从站
    kAdapterReady,    // SOEM 初始化完成，网卡已绑定
    kScanned,         // 从站扫描完成，已获取 SlaveInfo
    kMaintenance,     // 所有从站进入 PREOP，可执行 SDO 维护活动
    kOperational,     // 所有从站进入 OPERATIONAL，PDO 周期可运行
    kError            // 异常状态：只能 Stop() 回到 Uninitialized
};
```

> **设计原则**：状态只表示**稳定工况**，不表示一次性动作。`kPdoConfigured` / `kSafeOp` / `kDcConfigured` 等过渡步骤作为 `TransitionTo(kOperational)` 的内部子步骤，不暴露为 `ControllerState`。

当前阶段（PREOP + SDO）只走到 `kMaintenance`，不进入 `kOperational`。

## 3. 状态转换规则

### 3.1 允许的状态转换表

```cpp
static const std::map<ControllerState, std::vector<ControllerState>> kAllowedTransitions = {
    {kUninitialized,  {kAdapterReady}},
    {kAdapterReady,   {kScanned,        kUninitialized}},
    {kScanned,        {kMaintenance,    kUninitialized}},
    {kMaintenance,    {kOperational,    kUninitialized}},
    {kOperational,    {kUninitialized}},
    {kError,          {kUninitialized}}
};
```

表中记录**相邻或后退/特殊转换**。例如 `kOperational` 可以直接回到 `kUninitialized`（通过 `DoShutdown()` 逐级回滚）。

### 3.2 自动顺序前进

如果目标状态在当前状态之后，`TransitionTo()` 会按顺序执行中间步骤，每步都查表校验：

- `InitializeAdapter()` 后调用 `Scan()` → `EnterMaintenance()`：
  - `Uninitialized -> AdapterReady -> Scanned -> Maintenance`
- `StartOperation()` 调用 `TransitionTo(kOperational)`：
  - `Maintenance -> (内部 PdoConfigure -> SafeOp -> DcConfigure) -> Operational`

内部步骤不更新 `ControllerState`，只有最终到达 `kOperational` 时才更新状态。

### 3.3 非法转换

- 从 `kUninitialized` 直接跳到 `kOperational` 会被拒绝。
- 从 `kError` 只能调用 `Stop()` 回到 `kUninitialized`；其他目标都会报错。

## 4. 状态转换实现

| 方法 | 前置状态 | 作用 | 失败策略 |
|---|---|---|---|
| `DoInit()` | `kUninitialized` | `ecx_init()` 绑定网卡 | 保持 `kUninitialized` |
| `DoScan()` | `kAdapterReady` | `ecx_config_init()` 扫描从站，打印 SlaveInfo | 调用 `DoShutdown()` 回滚 |
| `DoEnterMaintenance()` | `kScanned` | 请求 PREOP 并确认，创建占位 SlaveNode | 调用 `DoShutdown()` 回滚 |
| `DoPdoConfigure()` | `kMaintenance` | 调用 `SlaveNodeManager::ConfigureAll()` 填充配置 | 进入 `kError` |
| `DoSafeOp()` | 内部过渡 | `ConfigureProcessData()` + 请求 SAFEOP 并确认 | 进入 `kError` |
| `DoDcConfigure()` | 内部过渡 | `ConfigureDc()` 配置分布式时钟 | 进入 `kError` |
| `DoOperational()` | 内部过渡 | 请求 OPERATIONAL 并确认 | 进入 `kError` |
| `DoShutdown()` | 任意 | 根据当前状态逐级回滚，最终关闭 SOEM | 总是成功 |
| `EnterErrorState()` | 任意 | 设置 `state_ = kError` 并记录原因 | — |

### 4.1 回滚路径

`DoShutdown()` 根据当前状态精确回滚：

```text
Operational -> SAFE_OP -> PRE_OP -> INIT
Maintenance -> INIT
Scanned / AdapterReady -> 直接 Close
Uninitialized -> 无操作
```

Maintenance 阶段停止时不会请求 SAFEOP，而是直接回到 INIT。

## 5. 公共接口

### 5.1 生命周期接口

```cpp
// 便捷接口：Uninitialized -> AdapterReady -> Scanned -> Maintenance
bool Initialize(const EcMasterConfig &config);

// 只初始化网卡/SOEM，到 AdapterReady
bool InitializeAdapter(const EcMasterConfig &config);

// AdapterReady -> Scanned
bool Scan();

// Scanned -> Maintenance
bool EnterMaintenance();

// Maintenance -> Operational（PDO 阶段使用）
bool StartOperation();

// 安全停止，回到 kUninitialized
void Stop();
```

### 5.2 状态查询接口

```cpp
ControllerState GetState() const;                          // 获取当前状态
size_t GetSlaveCount() const;                              // 获取扫描到的从站数
static const char *StateToString(ControllerState state);   // 状态转字符串
std::string GetLastError() const;                          // 获取最近一次错误原因

bool IsInitialized() const;  // state_ != kUninitialized
bool IsOperational() const;  // state_ == kOperational
```

### 5.3 周期与监控接口

```cpp
// 执行一次 PDO 周期（仅在 kOperational 有效）
void RunOneCycle();

// 状态监控：Maintenance/Operational 阶段检测到掉线则进入 kError
void CheckSlaveStates();
```

### 5.4 Activity 接口

```cpp
// 执行同步维护活动（第一阶段采用同步模型，后续可扩展异步）
bool ExecuteActivity(std::unique_ptr<EcatActivity> activity);

// 查询是否有活动正在执行
bool IsActivityRunning() const;
```

## 6. 使用示例

### 6.1 当前命令行交互用法

```bash
sudo ./ecat_scan enp0s31f6
```

启动后输入命令：

```text
scan
config
diagnose
inspect
stop
exit
```

对应的状态迁移：

```text
Uninitialized --InitializeAdapter--> AdapterReady --scan--> Scanned --config--> Maintenance
```

`diagnose` / `inspect` 等 Activity 在 `kMaintenance` 下执行，不改变 `ControllerState`。

### 6.2 编程式用法（后续 PDO 阶段）

```cpp
mo_ecat::EcatController controller;

// 走到 Maintenance
if (!controller.Initialize(config)) return 1;

// 可选：SDO 配置 PDO 映射

// 进入 OP
if (!controller.StartOperation()) return 1;
// controller.GetState() == kOperational

while (running) {
    controller.RunOneCycle();
    controller.CheckSlaveStates();
    if (!controller.IsOperational()) {
        // 进入 kError，需要 Stop() 后恢复
        break;
    }
}

controller.Stop();
```

## 7. 异常处理

### 7.1 初始化阶段异常

`DoInit()`、`DoScan()`、`DoEnterMaintenance()` 失败时调用 `DoShutdown()`，最终回到 `kUninitialized`，不进入 `kError`。

### 7.2 PDO 阶段异常

`StartOperation()` 内部任一子步骤（PDO 配置、SafeOp、DC 配置、Op）失败时进入 `kError`。此时只能调用 `Stop()` 回到 `kUninitialized`。

### 7.3 运行中掉线

`CheckSlaveStates()` 覆盖两个稳定状态：

```cpp
void EcatController::CheckSlaveStates()
{
    master_.CheckSlaveStates();

    switch (state_) {
    case ControllerState::kMaintenance:
        if (!master_.CheckAllSlavesInState(EC_STATE_PRE_OP)) {
            EnterErrorState("Slave dropped out of PREOP");
        }
        break;
    case ControllerState::kOperational:
        if (!master_.CheckAllSlavesInState(EC_STATE_OPERATIONAL)) {
            EnterErrorState("Slave dropped out of OPERATIONAL");
        }
        break;
    default:
        break;
    }
}
```

进入 `kError` 后：

- `RunOneCycle()` 拒绝执行。
- `StartOperation()` 等转换被拒绝。
- 必须 `Stop()` 回到 `kUninitialized` 后重新初始化。

## 8. 注意事项

1. **当前阶段只到 `kMaintenance`**：命令行不调用 `start` / `StartOperation()`，不会进入 PDO 阶段。
2. **占位节点**：`kMaintenance` 时 `SlaveNode` 只是占位对象，不分配 PDO buffer，不创建 `DeviceProfile`。
3. **Activity 不改变状态**：所有维护 Activity 在 `kMaintenance` 下执行，失败策略由 Activity 自己声明。
4. **状态查询**：优先使用 `IsInitialized()` / `IsOperational()` 或 `GetState()`，不要直接依赖内部 `state_`。
5. **日志**：关键状态转换点会打印日志；扫描后打印一次完整 SlaveInfo，进入 Maintenance 后只打印 summary。
6. **过渡步骤不暴露**：PDO 配置、SafeOp、DC 配置是 `StartOperation()` 的内部实现细节，不对外暴露为独立状态。

## 9. 后续可扩展点

- 增加 `start` 命令，调用 `StartOperation()` 进入 PDO 阶段。
- `CheckSlaveStates()` 返回更详细的异常信息（从站 id、name、状态码）。
- 增加自动降级/重试策略，例如从 `kError` 尝试回到 `kMaintenance` 而不是只能 Stop。
- 为 Activity 增加异步执行能力。
